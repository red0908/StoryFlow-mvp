# 创建角色模块设计文档：MBTI 心动相亲局

---

## 1. 设计目标与模块概述

### 1.1 目标
创建角色模块是玩家点击首页「开始游戏」后的第一个功能页面，负责收集玩家性别、年龄、职业、MBTI 与一句话介绍，完成个人角色创建并写入状态与本地存储，为后续「选择相亲对象」与剧情互动提供数据基础。设计上需与首页、视觉设计文档保持一致，保证流程顺畅、表单清晰、资源引用统一。

### 1.2 在整体流程中的位置
```
首页（开始游戏） → 创建角色 → 选择相亲对象（5选1） → 四章剧情 → 结局
```
- **入口**：首页底部按钮「开始游戏」点击后进入创建角色页面（或步骤条中的第一步）。
- **出口**：提交成功后进入「选择相亲对象」模块；若支持「使用上次角色」则可能跳过本页。

### 1.3 兼容范围
- **当前阶段**：优先兼容 **Web 端 PC**（桌面浏览器）。
- 后续可扩展移动端（表单单列、触控友好的滑条与选项）。

## 2. 需求来源与字段定义

### 2.1 依据需求文档（2.2 角色创建）

| 字段         | 内容/类型        | 说明 |
|--------------|------------------|------|
| 性别         | 男 / 女          | 用于匹配异性候选人（男→女性候选人，女→男性候选人） |
| 年龄         | 25～35 岁        | 滑动条或下拉选择，仅记录，剧情中可引用 |
| 职业         | 5 种预设         | 医生 / 教师 / 工程师 / 设计师 / 律师，影响初始介绍文本 |
| MBTI         | 四选一           | ENFJ / INTJ / INFJ / ENTJ，附带一句人格描述 |
| 一句话介绍   | 自由填写或系统生成 | 用于展示在候选人界面（展示「我」的信息） |

### 2.2 数据结构（与需求文档 5.2 一致）

```typescript
interface Player {
  gender: 'male' | 'female';
  age: number;           // 25～35
  job: string;           // 职业 ID，与 gameConfig.json 中 jobs[].id 一致
  mbti: 'ENFJ' | 'INTJ' | 'INFJ' | 'ENTJ';
  description: string;  // 一句话介绍
}
```

### 2.3 配置数据来源
- **职业与 MBTI 选项**：来自 `public/data/gameConfig.json`。
  - `jobs`：5 种职业的 `id`、`label`。
  - `mbti`：4 种人格的 `id`、`label`、`tagline`（如「热情的教育者」）。
- 前端加载该 JSON，驱动职业选择器与 MBTI 选择器的展示与提交值。

## 3. 资源路径与文件清单

### 3.1 根路径说明
路径均相对于项目 `public` 目录，前端引用使用 `/background/xxx`、`/person_img/xxx` 等形式。

### 3.2 背景资源（`/public/background/`）

| 文件名      | 用途说明 |
|-------------|----------|
| `bg-01.png` | 创建角色页全屏背景（可与首页共用，铺满视口 `background-size: cover`） |
| `card.png`  | 表单区块或卡片背景（与首页卡片风格统一） |
| `card1.png` | 备用卡片样式，可用于表单项分组或 MBTI 卡片底图 |

### 3.3 人物图片资源（`/public/person_img/`）

用于 **MBTI 选择区域** 的视觉展示（每类人格对应一张头像，便于玩家识别）：

| MBTI | 可选展示（示例） | 文件名（男/女按需） |
|------|------------------|----------------------|
| ENFJ | 男 | `avatar_male_enfj.png` |
| INTJ | 女 | `avatar_female_intj.png` |
| INFJ | 女 | `avatar_female_infj.png` |
| ENTJ | 男 | `avatar_male_entj.png` |

说明：创建角色时玩家仅选择「人格类型」，不选具体立绘；头像用于在 MBTI 选项卡片上展示人格氛围，可按设计选用 4 张（与首页四卡片一致）或 8 张按性别切换（见下文交互）。

### 3.4 配置与数据（`/public/data/`）

| 文件名             | 用途 |
|--------------------|------|
| `gameConfig.json`  | 职业列表 `jobs`、MBTI 列表 `mbti`（含 tagline），供表单选项与校验使用 |

## 4. 页面结构与布局

### 4.1 整体布局（PC）

- **视口**：以 1920×1080 为参考，可设最小宽度（如 1280px）避免过窄。
- **背景**：`bg-01.png` 铺满、居中覆盖。
- **内容区**：单列居中表单，可置于毛玻璃或 `card.png` 风格的容器内，与首页视觉统一。

### 4.2 自上而下区域划分

| 区域       | 内容说明 |
|------------|----------|
| 顶部标题   | 如「创建我的角色」或「认识一下自己」，主标题 + 可选副标题 |
| 表单区     | 性别 → 年龄 → 职业 → MBTI → 一句话介绍（顺序可微调） |
| 底部操作   | 「确认创建」主按钮；可选「随机填充」「使用上次角色」 |

### 4.3 各表单项设计要点（对齐视觉设计 4.2.2）

- **性别**
  - 两个选项：男 / 女。
  - 形式：圆形按钮或并排卡片，选中态填充主色（如粉色/酒红），与视觉文档一致。

- **年龄**
  - 范围：25～35。
  - 形式：滑动条（可自定义心形滑块）或下拉/步进器，实时显示当前数值。

- **职业**
  - 5 种：医生、教师、工程师、设计师、律师。
  - 形式：图标 + 文案，展示在选项旁；点击选中高亮；职业图标可来自视觉文档规范（60×60px 圆形图标），若暂无资源可先用文字 + 占位图。

- **MBTI**
  - 4 种：ENFJ、INTJ、INFJ、ENTJ。
  - 形式：四张卡片，每张展示对应代表色、图标（或 `person_img` 头像）、label + tagline；点击选中高亮。
  - 可与首页四卡片风格一致（如使用 `card.png` 或 `card1.png` 作底）。

- **一句话介绍**
  - 单行或多行输入框，带字数限制（如 20～50 字）；可选「根据 MBTI+职业 生成」按钮，生成后用户可编辑。

## 5. 交互与逻辑

### 5.1 首页「开始游戏」按钮

- **触发**：点击首页底部「开始游戏」按钮。
- **行为**：跳转或切换到「创建角色」页面（路由或条件渲染）。
- **状态**：不依赖后端；若本地已有角色且产品支持「快速开始」，可在进入前弹窗询问「新建角色」或「使用上次角色」。

### 5.2 创建角色页内交互

- **性别**：点选即更新，影响后续候选人性别（男→女候选人，女→男候选人）。
- **年龄/职业/MBTI**：点选或滑动即更新；MBTI 变更后可联动「一句话介绍」的生成建议（若有）。
- **一句话介绍**：
  - 必填或可选（按产品定）；若有「系统生成」，点击后根据当前职业 + MBTI 填充模板，用户可修改。
- **确认创建**：
  - 前端校验：年龄 25～35、职业与 MBTI 在配置内、介绍字数符合限制。
  - 通过后：写入 Redux（如 `playerSlice`）并写入 `localStorage`（需求 3.1），再跳转到「选择相亲对象」页面。

### 5.3 可选功能

- **随机填充**：除「一句话介绍」外，性别/年龄/职业/MBTI 随机赋值，便于测试或快速体验。
- **使用上次角色**：从 `localStorage` 读取最近一次 `Player`，若存在则弹窗确认后直接进入选择对象页。

### 5.4 校验规则建议

| 字段     | 规则 |
|----------|------|
| 性别     | 必选，male / female |
| 年龄     | 必选，25 ≤ value ≤ 35，整数 |
| 职业     | 必选，值在 gameConfig.jobs[].id 中 |
| MBTI     | 必选，值在 gameConfig.mbti[].id 中 |
| 一句话介绍 | 必填时：非空；字数上限如 50 字 |

---

## 6. 与状态管理及本地存储的衔接

- **Redux**：提交后 dispatch 更新 `playerSlice`（或等价 store），将完整 `Player` 写入 state，供选择对象页与剧情页读取（如显示「我」的 MBTI、职业、介绍）。
- **localStorage**：同需求文档 3.1，保存玩家最近一次角色信息；key 建议统一（如 `storyflow_player`），便于「使用上次角色」与刷新恢复。

## 8. 与现有设计的一致性

- **色彩与风格**：主色、毛玻璃、圆角、MBTI 代表色与《视觉设计》文档一致（ENFJ 暖橙、INTJ 冷紫、INFJ 森林绿、ENTJ 酒红）。
- **字体与组件**：标题、按钮、卡片与首页、视觉文档统一；按钮 hover/点击反馈（缩放、发光）保持一致。
- **资源复用**：背景、卡片、人物图与首页共用同一套 public 资源，避免重复资源。
