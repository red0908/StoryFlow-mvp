## 2. 产品概述与核心价值

### 2.1 产品定位

一款以 **MBTI 人格理论**为核心的轻量级**相亲互动游戏**。玩家在多个剧本中创建角色，通过选项驱动剧情，体验性格碰撞与情感发展。单机、纯前端，无后端依赖。

### 2.2 核心价值

- **双维度数值**：好感度（对方感受）+ 贴合度（自我认同），共同驱动剧情与结局。
- **多剧本入口**：剧本大厅选择剧本 → 创建角色 → 选择相亲对象（5 选 1）→ 四章剧情。
- **四种结局**：灵魂伴侣 / 表面和谐 / 知己难求 / 分道扬镳，由好感度与贴合度综合判定。
- **玩家档案与收集**：结局写入本地档案（历史记录、角色图鉴），心域地图展示收集进度；AI 评语 MVP 为模拟版。
- **沉浸式音频**：主菜单 BGM、剧情场景 BGM 切换、UI 音效（点击、好感/贴合变化），音量设置持久化。
- **移动端可用**：主要页面具备响应式布局与触控友好，手机浏览器可读可点。
- **线上可访问**：生产构建部署至 Vercel，支持 SPA 路由与直接访问。

## 3. 用户流程与功能模块

### 3.1 整体流程

```
首页 → 剧本大厅（选剧本）→ 创建角色 → 选择相亲对象（5 选 1）→
四章剧情（选项驱动，双数值）→ 结局（四型判定）→ 模拟 AI 评语 →
更新玩家档案 → 可进入心域地图/个人档案，或重新开始
```

### 3.2 路由与页面一览

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | HomePage | 首页：主标题、人物图、玩法介绍弹窗、开始游戏/玩家档案 |
| `/scripts` | ScriptHall | 剧本大厅：列表来自 index.json，选剧本后进创建角色 |
| `/profile` | ProfilePage | 个人档案：昵称、魅力、已解锁剧本等 |
| `/map` | HeartMapPage | 心域地图：MBTI 图鉴、历史记录（含 AI 评语） |
| `/create/myRole` | CreateRole | 创建角色：性别、年龄、职业、MBTI、一句话介绍 |
| `/blindDate` | BlindDatePage | 选择相亲对象：5 选 1，startGame → /game |
| `/game` | GamePage | 剧情：章节、节点、选项、双数值、效果/条件 |
| `/ending` | EndingPage | 结局：四型展示、模拟 AI 评语、写入档案 |
| `/select` | SelectPage | 占位 |

### 3.3 核心玩法要点

- **剧本**：`/data/scripts/index.json` 列表；单剧本 JSON 含 `id`、`title`、`nodes`、`characters` 等。
- **角色创建**：性别、年龄（25～35）、职业、MBTI（ENFJ/INTJ/INFJ/ENTJ）、一句话介绍。
- **候选人**：按剧本与 gameConfig 随机生成 5 位异性，选一位进入剧情。
- **剧情**：节点文本支持 `{opponent.name}`、`{player.description}` 等插值；选项带 effects（好感/贴合/职业/flag）与 condition（可见性）。
- **结局**：按好感度、贴合度判定四型；MVP 使用规则/模板生成 AI 评语并写入档案。

### 4.3 状态与数据流

- **usePlayerStore**：玩家角色（创建角色写入，剧情/结局读取）。
- **useGameStore**：对手、章节、节点、好感度、贴合度、flags；选对象时 startGame，剧情页读写，结局页读取后 resetGame。
- **useScriptStore**：当前剧本 id/summary；剧本大厅 setCurrentScript，创建/选人/剧情按 currentScriptId 加载剧本。
- **useProfileStore**：玩家档案（localStorage 持久化）；结局页 appendEnding、bumpEncounter；心域地图/个人档案只读展示。

## 5. 移动端适配（已完成）

### 5.1 适配原则

- 主要页面在手机浏览器上**可读、可点、不横向溢出**。
- 按钮与可点击区域满足**最小触控尺寸**（约 44px）。
- 兼顾**安全区**（刘海、底部横条），避免内容被遮挡。

### 5.2 全局设置

- **viewport**（`index.html`）  
  `width=device-width, initial-scale=1.0, maximum-scale=1.0, viewport-fit=cover`  
  保证移动端正确缩放与安全区扩展。

- **触控优化**（`src/index.less`）  
  在 `@media (hover: none) and (pointer: coarse)` 下为 `a, button` 设置 `touch-action: manipulation`，减少点击延迟。

### 5.3 响应式断点

各页面采用统一断点约定（与 Tailwind 可配合使用）：

| 断点 | 媒体查询 | 说明 |
|------|----------|------|
| 移动端 | `max-width: 767px` | 手机竖屏 |
| 平板 | `min-width: 768px` and `max-width: 1023px` | 平板竖屏/小屏横屏 |
| 桌面 | `min-width: 1024px` 或默认 | PC / 大屏 |

### 5.4 各页面适配要点

| 页面 | 适配要点（移动端 & 平板） |
|------|----------------------------|
| HomePage | 底部按钮纵向排列、全宽/最大宽度约束、触控最小高度 44px；玩法介绍弹窗自底部上滑、圆角、安全区 padding |
| ScriptHall | 剧本卡片单列或适配宽度，触控区域充足 |
| CreateRole | 表单项与按钮宽度、间距、最小触控高度 |
| BlindDatePage | 候选人卡片布局、按钮区域 |
| GamePage | 状态栏、剧情区、选项列表的宽度与字号，选项按钮触控友好 |
| EndingPage | 结局内容区、按钮区、安全区 |
| HeartMapPage | 图鉴与历史列表的布局与触控 |
| ProfilePage | 信息区与按钮布局 |

实现上，各页面在对应 `.less` 中通过 `@media (max-width: 767px)` 与 `@media (min-width: 768px) and (max-width: 1023px)` 覆盖布局、内边距、字体、按钮尺寸等；弹窗/底部栏使用 `env(safe-area-inset-*)` 预留安全区。

### 5.5 首页布局示例（文档化约定）

- 移动端：`home-footer` 使用 `flex-direction: column`、`max-width: 280px`，按钮 `min-height: 44px`、`width: 100%`。
- 玩法介绍弹窗：`max-height: calc(100vh - 2rem - env(safe-area-inset-bottom, 0px))`，`padding-bottom: env(safe-area-inset-bottom, 0)`，关闭按钮最小 44×44px。

## 6. 部署方案（Vercel）

### 6.1 构建与产物

- **开发**：`npm run dev`（默认端口 5173）。
- **生产构建**：`npm run build`，产出目录 `dist/`。
- **本地预览**：`npm run preview`。

### 6.2 Vercel 部署要点

- 将项目（或仅 `dist/` 的静态站点）部署到 Vercel。
- 若使用「连接 Git 仓库 + 构建命令」方式，构建命令为 `npm run build`，输出目录为 `dist`。
- **SPA 回退**：所有路径应回退到 `index.html`，避免刷新或直接访问子路径时 404。在项目根目录配置 `vercel.json`：

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

### 6.3 环境与域名

- 无环境变量硬编码敏感信息；剧本与配置均为静态 JSON。
- 部署完成后可通过 Vercel 分配的域名（及自定义域名）直接访问全站路由。

## 7. 数据与配置

### 7.1 剧本

- **列表**：`/data/scripts/index.json`，每项含 `id`、`title`、`description`、`recommendedMbti`、`cover`。
- **单剧本**：`/data/scripts/<id>.json` 或兼容的 `/data/story.json`，结构含 `nodes`、`characters` 等（见 `types.ts` 中 `Script`、`StoryNode`）。

### 7.2 配置

- **gameConfig.json**：`names`（male/female）、`jobs`、`mbti`，供创建角色与候选人生成使用。

### 7.3 本地持久化

- **玩家档案**：`localStorage` 键 `storyflow_profile`，结构见 `PlayerProfile`、`EndingRecord`、`CharacterRecord`。
- **音频设置**：`localStorage` 键 `storyflow_audio_settings`，存 BGM/音效/环境音音量及静音开关。

## 9. 附录：类型与结局判定摘要

- **MBTI**：`ENFJ` | `INTJ` | `INFJ` | `ENTJ`。
- **EndingType**：`success`（灵魂伴侣）| `surface`（表面和谐）| `friend`（知己难求）| `fail`（分道扬镳）。
- **结局判定**（逻辑见 EndingPage）：  
  灵魂伴侣 好感≥80 且 贴合≥70；表面和谐 好感≥80 且 贴合<50；知己难求 好感<60 且 贴合≥70；分道扬镳 好感<50 且 贴合<50；其余按规则就近归类。
