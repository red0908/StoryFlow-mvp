# StoryFlow 完整剧本案例大纲

## 1. 说明

- **用途**：作为 `public/data/story.json` 的编写依据，从第一章开头到第四章结束形成完整可玩剧情；结局由好感度在结局页判定，不单独设节点。
- **数据结构**：与代码中 `StoryNode`、`StoryOption`、`AffectionEffect`、`AffectionCondition` 一致（见技术架构文档 7.2、types.ts）。
- **规模**：4 章，共 14 个节点（章内 3~4 个），总节点数 ≤15；每节点 2~3 个选项。
- **插值**：正文与选项中的 `{opponent.name}`、`{opponent.mbti}` 等会在运行时被替换。

---

## 2. 章节与节点总览

| 章节 | 主题     | 节点 ID      | 场景概要 |
|------|----------|--------------|----------|
| 第一章 | 开头·初遇 | c1_node1～c1_node4 | 咖啡厅初次见面、寒暄、话题选择、结束离场 |
| 第二章 | 发展·再会 | c2_node1～c2_node4 | 公园/餐厅第二次约会、深入话题、小互动 |
| 第三章 | 高潮·心意 | c3_node1～c3_node3 | 关系推进或小矛盾、关键选择 |
| 第四章 | 结局·抉择 | c4_node1～c4_node3 | 表白/确认关系、收束；最后一节点无 nextNode → 进入结局页 |

## 3. 第一章·开头（初遇）

### c1_node1 — 咖啡厅见面

- **id**: `c1_node1`
- **chapter**: 1
- **text**: 你坐在咖啡厅，等待 {opponent.name} 的到来。她/他穿着一件简单的衬衫，微笑着向你打招呼，举手投足间带着一点 {opponent.mbti} 特有的气质。
- **options**:
  - 直接聊聊工作与日常。 → `c1_node2`，好感 +2（固定）
  - 先夸赞对方的穿着，拉近距离。 → `c1_node2`，valueByMbti（ENFJ+3，INTJ+1，INFJ+2，ENTJ+1）
  - 开个小玩笑缓和气氛。 → `c1_node2`，valueByMbti（ENFJ+2，INTJ+0，INFJ+1，ENTJ+2）

### c1_node2 — 话题打开

- **id**: `c1_node2`
- **chapter**: 1
- **text**: 你们聊开了。{opponent.name} 提到最近工作有点忙，但很享受现在的节奏。她/他似乎在观察你的反应。
- **options**:
  - 表示理解，分享自己最近的压力。 → `c1_node3`，好感 +2
  - 提议先点些喝的，再慢慢聊。 → `c1_node3`，valueByMbti（INTJ+2，ENFJ+1，INFJ+2，ENTJ+1）
  - 问对方业余时间喜欢做什么。 → `c1_node3`，valueByMbti（ENFJ+3，INTJ+1，INFJ+2，ENTJ+1）

### c1_node3 — 兴趣与价值观

- **id**: `c1_node3`
- **chapter**: 1
- **text**: 话题转到了爱好和周末安排。{opponent.name} 说起了自己喜欢的事，眼神里带着光。
- **options**:
  - 认真听并追问细节。 → `c1_node4`，好感 +3
  - 也分享自己的爱好，找共同点。 → `c1_node4`，valueByMbti（ENFJ+2，INTJ+2，INFJ+3，ENTJ+2）
  - 委婉表示今天时间有限，改天再聊。 → `c1_node4`，好感 -2

### c1_node4 — 第一章收尾

- **id**: `c1_node4`
- **chapter**: 1
- **text**: 咖啡见底，{opponent.name} 看了看时间，笑着说今天聊得很开心，有机会再约。
- **options**:
  - 主动提出下次由你请客，约好时间地点。 → `c2_node1`，好感 +3
  - 表示今天也很开心，等对方有空再联系。 → `c2_node1`，好感 +1
  - 只说「好的，再联系」，略显冷淡。 → `c2_node1`，好感 -1

## 4. 第二章·发展（再会）

### c2_node1 — 第二次约会开场

- **id**: `c2_node1`
- **chapter**: 2
- **text**: 周末，你们在公园门口碰面。{opponent.name} 换了一身休闲装，比上次更放松一些。
- **options**:
  - 夸对方今天看起来很有精神。 → `c2_node2`，valueByMbti（ENFJ+3，INTJ+1，INFJ+2，ENTJ+1）
  - 提议先随便走走，边走边聊。 → `c2_node2`，好感 +2
  - 问对方有没有特别想逛的地方。 → `c2_node2`，valueByMbti（ENFJ+2，INTJ+2，INFJ+2，ENTJ+2）

### c2_node2 — 散步中的对话

- **id**: `c2_node2`
- **chapter**: 2
- **text**: 走在林荫道上，{opponent.name} 聊起对未来的想法——工作、生活，还有对感情的期待。
- **options**:
  - 坦诚地说自己的想法，寻求共鸣。 → `c2_node3`，好感 +3
  - 更多倾听，偶尔回应。 → `c2_node3`，valueByMbti（INTJ+2，INFJ+3，ENFJ+2，ENTJ+1）
  - 开玩笑说「想那么多干嘛，先享受当下」。 → `c2_node3`，valueByMbti（ENFJ+2，INTJ+0，INFJ+0，ENTJ+1）

### c2_node3 — 小插曲

- **id**: `c2_node3`
- **chapter**: 2
- **text**: 路过一家小店，{opponent.name} 多看了几眼橱窗里的东西，又笑着说「看看就好」。
- **options**:
  - 提议进去看看，喜欢就买下来送她/他。 → `c2_node4`，valueByMbti（ENFJ+3，INTJ+0，INFJ+1，ENTJ+1）；若好感≥55 可选
  - 说「下次有机会再逛」，继续往前走。 → `c2_node4`，好感 +1
  - 也停下来认真看，聊几句品味。 → `c2_node4`，valueByMbti（INTJ+2，INFJ+2，ENFJ+1，ENTJ+2）

### c2_node4 — 第二章收尾

- **id**: `c2_node4`
- **chapter**: 2
- **text**: 傍晚时分，你们在公园长椅上坐了一会儿。{opponent.name} 说今天过得很愉快，希望还有下次。
- **options**:
  - 约定下次一起去吃饭或看电影。 → `c3_node1`，好感 +3
  - 说「今天谢谢你抽时间」，礼貌道别。 → `c3_node1`，好感 +1
  - 表达「和你在一起很放松」，略带暧昧。 → `c3_node1`，valueByMbti（ENFJ+3，INTJ+1，INFJ+2，ENTJ+2）；condition: affection ≥ 50

## 5. 第三章·高潮（心意）

### c3_node1 — 关系升温

- **id**: `c3_node1`
- **chapter**: 3
- **text**: 几次见面下来，你和 {opponent.name} 越来越熟。这次约在一家安静的餐厅，她/他主动问起你最近过得怎样。
- **options**:
  - 分享最近的开心事和烦恼，把对方当自己人。 → `c3_node2`，好感 +3
  - 反问对方，把话题引回她/他。 → `c3_node2`，valueByMbti（INFJ+3，ENFJ+2，INTJ+2，ENTJ+1）
  - 轻描淡写带过，保持一点距离。 → `c3_node2`，好感 -1

### c3_node2 — 分歧或默契

- **id**: `c3_node2`
- **chapter**: 3
- **text**: 聊到对「理想关系」的看法时，{opponent.name} 很认真地说了一通。你们有些地方一致，也有些不同。
- **options**:
  - 表示「不同很正常，重要的是愿意互相理解」。 → `c3_node3`，好感 +3
  - 坚持自己的看法，但语气温和。 → `c3_node3`，valueByMbti（INTJ+2，ENTJ+2，ENFJ+0，INFJ+1）
  - 岔开话题，避免深入。 → `c3_node3`，好感 -2

### c3_node3 — 第三章收尾

- **id**: `c3_node3`
- **chapter**: 3
- **text**: 晚餐结束，{opponent.name} 送你到门口，欲言又止。夜色里，你感觉彼此之间只差一句明确的话。
- **options**:
  - 主动说「下次我想正式请你吃顿饭，只有我们两个」。 → `c4_node1`，好感 +3
  - 说「今天谢谢，路上小心」，等对方先开口。 → `c4_node1`，好感 +1
  - 若好感 ≥ 60：直接问「你愿意和我试试看吗？」。 → `c4_node1`，好感 +2；condition: affection ≥ 60

## 6. 第四章·结局前（抉择）

### c4_node1 — 最后一步

- **id**: `c4_node1`
- **chapter**: 4
- **text**: 你们又一次见面。{opponent.name} 比以往都放松，像是已经把你当成了很重要的人。气氛正好。
- **options**:
  - 郑重表白，希望和对方认真交往。 → `c4_node2`，好感 +4
  - 用行动表达（如准备小礼物或惊喜），再问对方心意。 → `c4_node2`，valueByMbti（ENFJ+3，INFJ+3，INTJ+2，ENTJ+2）
  - 再观察一段时间，只说「能认识你真好」。 → `c4_node2`，好感 +0 或 -1

### c4_node2 — 对方的回应

- **id**: `c4_node2`
- **chapter**: 4
- **text**: {opponent.name} 沉默了几秒，然后笑了。她/他说，其实也在等一个合适的时机，把心里话说出来。
- **options**:
  - 牵起对方的手，说「那从今天开始吧」。 → `c4_node3`，好感 +3
  - 笑着说「还好我们都等到了」。 → `c4_node3`，好感 +2
  - 有点紧张，只说「谢谢你的答案」。 → `c4_node3`，好感 +1

### c4_node3 — 终章（无 nextNode，进入结局页）

- **id**: `c4_node3`
- **chapter**: 4
- **text**: 你们一起走了一段路，没有说太多话，但心里都清楚——这段缘分，已经开始了。剩下的，就交给时间和彼此的选择吧。
- **options**:
  - 「一起走下去吧。」 → 无 nextNode（或 nextNode 为空），好感 +2；点击后由游戏跳转 `/ending`，按当前 affection 判定结局。
  - 「今天谢谢你。」 → 无 nextNode，好感 +1；同上。
  - 「下次见。」 → 无 nextNode，好感 +0；同上。

**说明**：c4_node3 的三个选项的 `nextNode` 均不指向任何节点（或设为空字符串）。剧情引擎在「无下一节点」时执行 `navigate('/ending')`，由结局页根据 `affection` 显示「灵魂伴侣」「略有遗憾」或「分道扬镳」。

## 7. 结局判定（在结局页实现，非剧本节点）

| 好感度范围 | 结局类型   | 标题       |
|------------|------------|------------|
| affection ≥ 80 | success | 灵魂伴侣 |
| 50 ≤ affection < 80 | regret | 略有遗憾 |
| affection < 50 | fail | 分道扬镳 |

## 9. 选项效果与条件汇总（写作参考）

- **固定好感**：`effects: [{ type: "affection", operator: "add", value: 2 }]`
- **按 MBTI**：`effects: [{ type: "affection", operator: "add", valueByMbti: { ENFJ: 3, INTJ: 1, INFJ: 2, ENTJ: 1 } }]`
- **条件**：`condition: { type: "affection", operator: ">=", value: 60 }` 表示仅当好感 ≥60 时显示该选项。
- **减好感**：`operator: "subtract"`，配合 `value` 或 `valueByMbti`。

将本大纲中的各节点按上述格式填入 `story.json` 即可得到完整可玩剧本；正文中的 `{opponent.name}`、`{opponent.mbti}` 会在运行时由引擎替换为当前对象信息。
