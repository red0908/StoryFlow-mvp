## 📅 Day 3（第 3 天）：结局系统、玩家档案与 AI 评语模拟

### 目标
实现双数值结局判定，将游戏结果保存到玩家档案，并显示模拟 AI 评语。

### 任务清单
| 任务 | 详细说明 | 优先级 | 预计耗时 |
|------|----------|--------|----------|
| 1. 结局判定逻辑 | 在 `EndingPage` 中根据好感度、贴合度综合判定结局类型（灵魂伴侣/表面和谐/知己难求/分道扬镳），显示对应标题与描述 | P0 | 1h |
| 2. 玩家档案 Store | 创建 `useProfileStore`，定义 `PlayerProfile` 接口，实现 `localStorage` 读写（包含角色图鉴、历史记录、勋章） | P0 | 1.5h |

## 📅 Day 4（第 4 天）：音效集成与移动端适配

### 目标
为游戏添加背景音乐和 UI 音效，并确保在手机屏幕上正常显示。

## 🚨 风险与应对
| 风险 | 应对 |
|------|------|
| 音频自动播放被浏览器阻止 | 所有音频播放放在用户手势后（如点击“开始游戏”后），或在首页添加“点击进入”按钮触发音频初始化 |
| 剧本 JSON 转换出错 | 使用在线工具（Excel 转 JSON）或手动核对，先保证主线节点正确，支线可暂缓 |
| 移动端样式错乱 | 使用 Chrome 开发者工具模拟 iPhone SE 调试，优先保证核心页面可读 |
| Vercel 部署后刷新 404 | 在项目根目录创建 `vercel.json`，配置 `{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }` |
