# 简化版执行计划：MBTI 相亲互动游戏 MVP（兼职友好版）

考虑到您工作日只能投入 2 小时/天，周末 8 小时/天，我们将产品精简为核心功能，并重新规划步骤。目标是在 **两周内（共约 26 工时）** 上线一个可玩的简化版本，后续再迭代完善。

---

## 简化版 MVP 核心功能
- **角色创建**：性别、年龄、职业（5 选1）、MBTI（4 选1），一句话介绍可选。
- **候选人生成**：随机 5 位异性，展示姓名、年龄、职业、MBTI、一句话介绍（用文字，无头像）。
- **剧情**：4 章，每章 3~4 个节点，总节点数 ≤ 15。选项影响好感度，最终根据好感度触发 3 种结局。
- **技术**：纯前端，Vue 3 + Pinia + Tailwind，剧本 JSON 驱动，无本地存储（每次新游戏），无复杂条件分支，无表情切换，无动画特效（仅基础 CSS 过渡）。
- **视觉**：先用占位色块和免费图标，上线前用 AIGC 替换少量关键图（可选）。

---

## 总体时间规划（总工时约 26 小时）
- **第一个周末**（周六 8h + 周日 8h）：搭建完整可玩的核心框架（角色创建 → 选人 → 剧情 → 结局）。
- **工作日五天**（每天 2h）：填充剧本内容、测试、简单优化。
- **第二个周末**（8h）：视觉资源替换、部署上线、最终测试。

## 详细执行步骤（按天分配）

### 🟢 第一天（周六）：基础框架搭建（8 小时）

| 时间 | 用户任务 | Cursor 任务 |
|------|----------|-------------|
| **上午 4h** | ① 确定项目名称，创建 Git 仓库（可选）。<br>② 在本地创建 `story.json`，按模板填入 12 个空节点结构（只需 id、chapter、空 text、空 options）。<br>③ 准备姓名库（男/女各 5 个）、职业列表（医生、教师、工程师、设计师、律师）、MBTI 列表（ENFJ/INTJ/INFJ/ENTJ）。 | ① 初始化 Vite + Vue 3 + TypeScript + Pinia + Tailwind。<br>② 创建类型定义（`Player`、`Candidate`、`GameState`、`StoryNode`）。<br>③ 创建 Pinia store：`usePlayerStore`、`useGameStore`（含好感度、当前节点、对方角色）。 |
| **下午 4h** | ① 用 Midjourney 或免费图标网站准备几张占位图（可选，也可直接用色块）。<br>② 思考第一章第一个节点的文本（可先不写）。 | ① 实现角色创建界面（`CreateCharacter.vue`）：性别单选、年龄滑条、职业下拉/MBTI 卡片（用文字选择，无图标）。<br>② 实现候选人生成界面（`SelectCandidate.vue`）：根据玩家性别随机生成 5 个候选人，展示为简单卡片（姓名、年龄、职业、MBTI、一句话介绍随机生成）。 |

### 🟢 第二天（周日）：剧情引擎核心（8 小时）

| 时间 | 用户任务 | Cursor 任务 |
|------|----------|-------------|
| **上午 4h** | ① 为候选人一句话介绍编写简单生成规则（如 `{name}，{age}岁，{job}，{mbti}性格`）。<br>② 确认好感度初始值（50），结局阈值（≥80 完美，50~79 遗憾，<50 失败）。 | ① 实现剧情主界面（`Game.vue`）：顶部显示章节名、双方 MBTI 文字、好感度进度条（简单条形）。<br>② 实现剧本加载函数，根据当前节点 ID 渲染文本（支持变量替换 `{opponent.name}` 等）。<br>③ 底部选项区动态生成按钮（最多 3 个）。 |
| **下午 4h** | ① 填写第一章 3~4 个节点的文本和选项（可先用占位文本，如“选项 A”“选项 B”）。 | ① 实现选项点击处理：执行好感度增减（固定值），根据 `nextNode` 跳转。<br>② 当 `nextNode` 为空时，根据好感度范围跳转至结局界面。<br>③ 创建简单的结局界面（`Ending.vue`）：显示结局名称和评语，提供“重新开始”按钮。 |

**周日结束时，您已经拥有一个可玩的完整流程**（角色创建 → 选人 → 第一章剧情 → 结局），虽然剧本内容不完整，但核心机制已通。

### 🟡 工作日 1~5：内容填充与测试（每天 2 小时，共 10 小时）

#### 周一（2h）
- **用户**：编写第一章完整剧本（4 个节点，每个节点 2 个选项，指定好感效果）。

#### 周二（2h）
- **用户**：编写第二章剧本（4 节点）。
- **Cursor**：优化好感度进度条样式，确保移动端显示正常。

#### 周三（2h）
- **用户**：编写第三章剧本（4 节点）。
- **Cursor**：添加简单的选项条件（可选，如好感 >60 才出现某个选项，可用 `if` 简单实现）。

#### 周四（2h）
- **用户**：编写第四章剧本（4 节点），并确定各结局触发的具体好感阈值。
- **Cursor**：完善结局界面，显示双方 MBTI 和简短匹配分析。

#### 周五（2h）
- **用户**：用 AIGC 生成少量视觉资源（如一张咖啡厅背景图、5 个头像占位图），放入 `public` 文件夹。
- **Cursor**：替换背景图和头像，调整界面颜色，添加简单按钮悬停效果。

### 🟢 第二个周末（8 小时）：视觉优化与部署

#### 周六（4h）

- **Cursor**：将所有视觉资源集成到游戏中，按章节切换背景，按 MBTI 显示不同颜色标签。

#### 周日（4h）
- **用户**：全面测试所有分支，记录 bug。
- **Cursor**：修复 bug，配置 Vercel 部署，生成二维码。
- **共同**：上线并分享给朋友测试。

## 简化要点说明（与原始计划对比）
| 原计划 | 简化版 |
|--------|--------|
| 20 个节点 | 12~15 个节点 |
| 精致日系 UI，大量图标、立绘、表情 | 先用文字和色块，上线前替换少量图 |
| 复杂条件分支、flag 变量 | 仅好感度影响结局 |
| 本地存储、历史记录 | 无（每次新游戏） |
| 表情切换、动画特效 | 无（仅基础过渡） |
| 5 个候选人各 3 种表情 | 每个候选人只用一张占位图 |
| AIGC 生成全部资源（耗时） | 周末集中生成核心资源 |
