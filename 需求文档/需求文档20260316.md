# 产品需求设计文档：心域奇旅 · 相亲互动游戏（v2.1）

## 1. 产品概述
### 1.1 产品背景
一款以 MBTI 人格理论为核心的轻量级相亲互动游戏，玩家可在多个剧本中扮演不同角色，通过选项驱动剧情，体验性格碰撞与情感发展。v2.1 在 v2.0 基础上引入**音频系统**，通过背景音乐、环境音效与 UI 音效增强沉浸感。

### 1.2 核心价值
- **深度人格互动**：每个选项同时影响「好感度」（对方感受）和「贴合度」（自我认同），双维度驱动剧情分支。
- **多剧本扩展**：支持多套独立剧本（现代都市、古风、职场等），玩家可自由选择，剧本由 JSON 独立配置。
- **AI 智能总结**：结局时调用 AI 接口，根据玩家选择生成个性化情感分析报告，增强仪式感与分享欲。
- **成长收集系统**：勋章墙、攻略地图、AI 历史评价永久保存，激励玩家探索所有人格与结局。
- **沉浸式音频**：场景化背景音乐、环境音效与即时 UI 反馈音，让剧情体验更生动。

## 2. 用户流程与核心玩法
### 2.1 整体流程
```
启动游戏 → 选择剧本 → 创建个人角色 → 浏览异性候选人（5选1） → 
进入四章剧情（选项驱动） → 达成结局 → AI 生成评价报告 → 
更新玩家档案（解锁勋章/记录） → 返回主菜单或重新开始
```
- 剧本独立，每个剧本包含完整四章剧情（约20节点）及专属角色库。
- 结局判定综合「好感度」与「贴合度」，产生 4 种基础结局 + 隐藏结局。

### 2.2 剧本选择模块
- 主界面新增「剧本大厅」，展示已解锁/未解锁剧本（初始默认解锁 1 个，其余可通过达成特定条件解锁）。
- 每个剧本包含：
  - 封面图、名称、简介、推荐人格、攻略角色预览。
  - 剧本数据独立存储于 `/data/scripts/` 下（如 `script_modern.json`）。

### 2.3 角色创建（继承原有，但绑定当前剧本）
| 字段 | 内容 | 说明 |
|------|------|------|
| 性别 | 男 / 女 | 用于匹配异性候选人 |
| 年龄 | 25~35 岁（滑动条/下拉） | 可被剧本文本引用 |
| 职业 | 5 种预设（医生/教师/工程师/设计师/律师） | 影响选项效果与隐藏节点 |
| MBTI | 从 ENFJ/INTJ/INFJ/ENTJ 中单选 | 核心人格属性 |
| 一句话介绍 | 自由填写或系统生成 | 用于候选人界面展示 |

### 2.4 选择相亲对象
- 根据当前剧本的候选人库，随机生成 5 位异性候选人（性别与玩家相反）。
- 候选人包含：姓名、年龄、职业、MBTI、一句话介绍（结合剧本背景生成）。
- 玩家点击选择后，锁定该角色为本次相亲对象，进入剧情。

### 2.5 剧情互动（双数值系统）
- **顶部状态栏**：显示当前章节、双方 MBTI 图标、好感度（0-100）与贴合度（0-100）。
- **中部剧情框**：显示节点文本，支持动态插值 `{player.name}`、`{opponent.mbti}`、`{player.job}` 等。
- **底部选项区**：每个选项显示 2~4 个按钮，点击后触发效果：
  - 好感度变化（固定值或基于对方 MBTI 的映射）
  - 贴合度变化（基于玩家 MBTI 的映射）
  - 职业额外加成（若玩家职业匹配）
  - 设置/检查剧情 flag
- 选项可出现条件：好感度阈值、贴合度阈值、玩家 MBTI、玩家职业、flag 状态。

### 2.6 结局系统
- 第四章最后一个节点结束后，根据「好感度」与「贴合度」综合判定结局类型：
  | 结局 | 好感度 | 贴合度 | 说明 |
  |------|--------|--------|------|
  | 灵魂伴侣 | ≥80 | ≥70 | 双高，真正契合 |
  | 表面和谐 | ≥80 | <50 | 讨好对方，失去自我 |
  | 知己难求 | <60 | ≥70 | 三观合但不来电 |
  | 分道扬镳 | <50 | <50 | 完全不合 |
  | 隐藏结局 | 特殊 flag 组合 | 如“事业伙伴”“欢喜冤家”等 |
- 结局展示后，立即调用 **AI 评价模块**，生成个性化总结报告。

### 2.7 AI 评价模块
- 在结局页增加「生成 AI 情感报告」按钮（或自动触发）。
- 前端将本次游戏的关键数据打包发送至 AI 接口（或本地模拟）：
  - 玩家 MBTI、职业
  - 对方 MBTI、职业
  - 好感度、贴合度变化曲线（可简化为关键节点选择）
  - 达成结局类型
  - 触发过的隐藏节点
- AI 返回一段 100~200 字的评语，包含：
  - 对本次关系的总结
  - 对玩家选择风格的洞察
  - 一句温暖/幽默的寄语
- 评语可保存至玩家档案，并生成分享卡片。

### 2.8 玩家攻略地图（个人档案）
在主菜单新增「心域地图」入口，展示玩家所有游戏历史与收集进度：

- **MBTI 角色图鉴**：每个可攻略人格类型一个格子，显示：
  - 已攻略次数、最高好感度、解锁结局数
  - 点击可查看与该人格的所有历史记录（含 AI 评价）
- **剧本解锁地图**：以树状图或卡片形式展示所有剧本，标记完成度（每章通关/全结局达成）。
- **勋章墙**：
  - 人格勋章：首次与某 MBTI 达成「灵魂伴侣」
  - 职业勋章：以特定职业通关剧本
  - 探索勋章：触发所有隐藏节点
  - 连续游戏勋章：累计游戏次数等
- **AI 历史评价记录**：按时间倒序展示所有结局的 AI 评语，可点赞或分享。

## 3. 功能需求详情
### 3.1 初始化与数据持久化
- 使用 `localStorage` 存储：
  - 玩家个人档案（含勋章、攻略记录、AI 评价列表）
  - 最近一次游戏的角色信息与进度（可选）
- 玩家档案结构示例：
```typescript
interface PlayerProfile {
  charmLevel: number;          // 魅力等级（累计经验）
  charmExp: number;
  unlockedScripts: string[];   // 已解锁剧本ID
  characterCollection: {       // 角色图鉴
    [mbti: string]: {
      encounterCount: number;
      soulmateCount: number;
      endings: string[];        // 达成的结局ID列表
      bestAffection: number;
      bestAlignment: number;
    }
  };
  endingHistory: Array<{        // 历史记录（含AI评语）
    date: string;
    scriptId: string;
    opponentMbti: string;
    endingType: string;
    affection: number;
    alignment: number;
    aiComment: string;
  }>;
  badges: string[];             // 已获得勋章ID
}
```

### 3.2 剧本管理
- 剧本文件存放在 `public/data/scripts/`，每个剧本一个 JSON，格式与原有 `story.json` 兼容，但增加元数据：
```json
{
  "id": "modern_love",
  "title": "都市迷情",
  "description": "在繁华都市中寻找真爱...",
  "recommendedMbti": ["ENFJ", "ENTJ"],
  "characters": ["ENFJ", "INTJ", "INFJ", "ENTJ"],
  "nodes": [ ... ]  // 同原有story.json结构
}
```
- 剧本选择界面读取 `/data/scripts/index.json` 获取剧本列表（含解锁条件）。

### 3.3 剧本引擎增强
- 支持 `alignment` 效果：在选项 `effects` 中增加 `{ type: "alignment", valueByPlayerMbti: { ... } }`。
- 支持职业条件：在 `condition` 中增加 `playerJob` 判断，如 `{ "type": "playerJob", "value": "医生" }`。
- 支持贴合度条件：`{ "type": "alignment", "operator": ">=", "value": 60 }`。
- 效果执行器需同时更新 `affection` 和 `alignment`，并处理职业加成（通过 `jobEffect` 字段）。

### 3.4 AI 评价接口
- 初期可使用模拟数据：前端根据规则生成模板化评语。
- 后续接入真实 AI API（如 OpenAI），发送结构化数据，接收评语。
- 接口需考虑限流与错误处理，失败时显示默认评语。

### 3.5 攻略地图界面
- 采用网格布局展示 MBTI 图鉴，每个格子显示人格图标、攻略次数、结局数。
- 点击进入详情页，展示历史记录卡片（含 AI 评语）。
- 勋章墙以徽章形式陈列，悬停显示说明。

## 4. 系统架构
### 4.1 总体架构图
```
[前端 SPA] (React + Zustand)
  ├── 视图层：剧本大厅 / 角色创建 / 候选人选择 / 剧情互动 / 结局展示 / 心域地图
  ├── 状态管理：
  │   ├── usePlayerStore   (当前游戏角色)
  │   ├── useGameStore     (当前游戏进度：对手、节点、好感、贴合、flags)
  │   └── useProfileStore  (玩家永久档案，读写 localStorage)
  ├── 剧本引擎：剧本加载器、条件解释器、效果执行器、AI评价适配器
  ├── 音频管理器：Howler.js 封装，负责背景音乐、环境音效、UI音效的播放与控制
  └── 工具层：随机生成器、本地存储封装、AI 接口封装
```
- **无后端**，所有数据静态打包，AI 接口可选前端模拟或调用第三方 API（需跨域代理）。

### 4.2 模块划分
| 模块 | 职责 |
|------|------|
| 剧本大厅 | 展示剧本列表，处理解锁逻辑，加载选定剧本 |
| 角色创建 | 收集玩家信息，存入 usePlayerStore |
| 候选人模块 | 根据当前剧本配置生成 5 位异性候选人 |
| 剧情主控 | 加载剧本节点，渲染，处理选项，更新双数值与 flags |
| 结局模块 | 判定结局，触发 AI 评价，更新玩家档案 |
| 心域地图 | 读取玩家档案，展示收集进度与历史记录 |
| AI 评价适配器 | 封装 AI 调用逻辑，支持模拟与真实 API 切换 |
| 音频管理器 | 预加载音频，根据场景播放/停止/切换音轨，提供音量控制 |
| 本地存储 | 读写 useProfileStore 与游戏进度 |

## 5. 技术方案
### 5.1 技术栈（更新）
- **构建工具**：Vite
- **框架**：React 18 + TypeScript
- **状态管理**：Zustand（分三个 store）
- **样式**：Tailwind CSS + Less（页面级样式）
- **路由**：React Router 7（管理多页面：首页/剧本大厅/创建角色/选择对象/游戏/结局/地图）
- **音频引擎**：Howler.js（轻量、多音轨管理、支持 Web Audio）
- **AI 集成**：可选 axios 调用 API，或使用规则引擎模拟
- **部署**：Vercel / Netlify（静态托管）

### 5.2 关键数据结构扩展
#### 玩家档案 (PlayerProfile)
```typescript
interface PlayerProfile {
  charmLevel: number;
  charmExp: number;
  unlockedScripts: string[];
  characterCollection: Record<string, CharacterRecord>;
  endingHistory: EndingRecord[];
  badges: string[];
}
```

#### 游戏状态 (GameState) 扩展
```typescript
interface GameState {
  // ...原有字段
  alignment: number;           // 新增贴合度 0-100
}
```

#### 剧本节点选项效果扩展
```typescript
type Effect = 
  | { type: "affection"; value: number }
  | { type: "affection"; valueByMbti: Record<MBTI, number> }
  | { type: "alignment"; valueByPlayerMbti: Record<MBTI, number> }
  | { type: "setFlag"; key: string; value: any }
  | { type: "jobEffect"; job: string; affection?: number; alignment?: number };
```

### 5.3 条件解释器扩展
支持条件类型：
- `affection`（好感比较）
- `alignment`（贴合度比较）
- `playerMbti`（玩家 MBTI 是否等于某值）
- `playerJob`（玩家职业是否等于某值）
- `flag`（自定义变量）

## 7. 音频与音效设计
### 7.1 设计目标
- **增强代入感**：通过背景音乐和环境音让场景“活”起来。
- **即时反馈**：用简短的 UI 音效让操作手感更舒适，数值变化可听见。
- **氛围烘托**：根据剧情情绪（轻松、紧张、暧昧）切换音乐，强化情感波动。
- **轻量实现**：所有音频文件采用 MP3 格式，总大小控制在几 MB 内，通过 Howler.js 管理。

### 7.2 音频类型与场景匹配
#### 背景音乐 (BGM)
| 场景/状态 | 风格建议 | 循环 | 优先级 |
|----------|----------|------|--------|
| 主菜单 / 剧本大厅 | 轻快、温馨、略带期待感的钢琴或原声吉他 | 是 | P0 |
| 角色创建界面 | 中性、舒缓的背景，不干扰注意力 | 是 | P1 |
| 咖啡厅（第一章） | 爵士钢琴或轻爵士，带点慵懒感 | 是 | P0 |
| 公园（第二章） | 轻音乐+鸟鸣/风声等自然采样 | 是 | P0 |
| 餐厅（第三章） | 温柔的弦乐或钢琴独奏，略带浪漫 | 是 | P0 |
| 结局画面 | 根据结局不同：完美→温暖宏大；遗憾→轻柔忧伤；失败→平静或略带失落 | 是 | P1 |
| 隐藏节点触发 | 短暂的神秘/惊喜音效 + 恢复原有 BGM | 否 | P2 |

#### 环境音效 (Ambient SFX)
| 场景 | 音效 | 循环 | 优先级 |
|------|------|------|--------|
| 咖啡厅 | 咖啡机蒸汽声、杯碟轻碰声、远处低语 | 是 | P1 |
| 公园 | 鸟鸣、风声、树叶沙沙声、远处儿童嬉笑声 | 是 | P1 |
| 餐厅 | 餐具轻碰、柔和的背景人声（非常轻） | 是 | P1 |

#### UI 音效
| 交互 | 音效 | 优先级 |
|------|------|--------|
| 按钮点击 | 清脆的“咔哒”声或纸片翻动声 | P0 |
| 好感度增加 | 向上的音效（如叮~或清脆的铃声） | P0 |
| 好感度减少 | 向下的音效（如短暂的玻璃碎裂或低沉音） | P0 |
| 贴合度变化 | 不同的提示音（例如“叮咚”代表贴合） | P1 |
| 解锁隐藏节点 | 短暂的魔法/星光音效 | P1 |
| 翻页/章节切换 | 翻书声或轻快的过渡音 | P1 |
| 按钮悬停 | 轻微的“滴”声或点击前提示 | P2（可选） |
| 选项出现 | 柔和的“浮现”音（如气泡音） | P2 |

### 7.3 技术实现
- **音频管理器**：使用 Howler.js 封装，提供 `playBGM(id, fadeIn)`、`playSFX(id)`、`stopBGM(fadeOut)`、`setVolume(volume, type)` 等方法。
- **预加载**：进入游戏主菜单后，开始静默加载所有音频文件，避免游戏中卡顿。
- **场景驱动**：剧情主控模块根据当前节点的 `scene` 字段，通知音频管理器切换背景音乐和环境音。
- **音量控制**：在设置面板中提供 BGM、音效、环境音独立音量滑块及静音开关。
- **文件存放**：`public/audio/` 下按类型分文件夹（bgm/, sfx/, ambient/），文件命名与场景/效果对应。

### 7.4 MVP 音频开发优先级
| 优先级 | 内容 |
|--------|------|
| P0 | 主菜单 BGM、咖啡厅 BGM、公园 BGM、餐厅 BGM；按钮点击音效；好感度增减音效 |
| P1 | 环境音（咖啡厅/公园/餐厅）；贴合度变化音效；章节切换音效；隐藏节点触发音效；结局短音乐 |
| P2 | 按钮悬停音效；选项出现音效；旁白（可选） |

## 9. 风险与应对
| 风险 | 应对 |
|------|------|
| AI 接口不稳定或费用高 | 前期使用规则模板模拟，后期可选接入，并增加缓存 |
| 多剧本导致剧本编写量激增 | 先完成两个核心剧本，后续通过社区或众筹扩充 |
| 双数值系统可能让玩家困惑 | 界面明确显示两个进度条，结局时给出详细解释 |
| 本地存储空间限制 | 只保存最近 20 条历史记录，老记录自动清理 |
| 音频文件体积过大影响加载 | 使用 128kbps MP3 压缩，预加载时异步进行，不阻塞主流程 |
