**文档状态**：定稿（MVP 本周五上线）  
**版本**：2.2  
**日期**：2026-03-17

## 1. 产品概述

### 1.1 产品背景

一款以 MBTI 人格理论为核心的轻量级相亲互动游戏。玩家在多个剧本中扮演不同角色，通过选项驱动剧情，体验性格碰撞与情感发展。v2.2 在既有能力上收敛为「本周五上线 MVP」的可实现范围，明确必做与可延后项。

### 1.2 核心价值

- **双维度数值**：好感度（对方感受）+ 贴合度（自我认同），共同驱动剧情与结局。
- **多剧本入口**：剧本大厅选择剧本，创建角色后进入对应剧本的候选人选择与四章剧情。
- **四种结局**：灵魂伴侣 / 表面和谐 / 知己难求 / 分道扬镳，由好感度与贴合度综合判定。
- **玩家档案与收集**：结局写入本地档案（历史记录、角色图鉴），心域地图展示收集进度；AI 评语采用模拟版。
- **沉浸式音频（MVP 必做）**：主菜单 BGM、剧情场景 BGM 切换、基础 UI 音效（点击、好感/贴合变化），音量设置持久化。

## 2. 用户流程与核心玩法

### 2.1 整体流程

```
首页 → 剧本大厅（选剧本）→ 创建角色 → 选择相亲对象（5 选 1）→
四章剧情（选项驱动，双数值）→ 结局（四型判定）→ AI 评语（模拟）→
更新玩家档案 → 可进入心域地图/个人档案，或重新开始
```

### 2.2 剧本大厅

- 主入口：首页「开始游戏」进入剧本大厅。
- 展示剧本列表（来自 `/data/scripts/index.json`）：封面、标题、简介、推荐 MBTI。
- 点击剧本卡片后进入创建角色，并将所选剧本 ID 写入 store（或 URL），后续候选人与剧情均基于该剧本。

### 2.3 角色创建

| 字段         | 说明 |
|--------------|------|
| 性别         | 男/女，用于匹配异性候选人 |
| 年龄         | 25～35 岁 |
| 职业         | 预设职业（与 gameConfig 一致） |
| MBTI         | ENFJ/INTJ/INFJ/ENTJ 四选一 |
| 一句话介绍   | 自由填写或占位 |

### 2.4 选择相亲对象

- 根据当前剧本的候选人库（剧本 JSON 的 `characters` 等）随机生成 5 位异性候选人。
- 候选人含姓名、年龄、职业、MBTI、一句话介绍。选择后锁定为本次相亲对象，进入剧情。

### 2.5 剧情互动

- **顶部状态栏**：章节、双方 MBTI 图标、好感度与贴合度（0～100）。
- **剧情区**：节点文本，支持 `{opponent.name}`、`{opponent.mbti}`、`{player.description}` 等插值。
- **选项**：每个选项可带 effects（好感度/贴合度/职业加成/flag）与 condition（好感度、贴合度、玩家 MBTI、职业、flag）。选项可见性由条件控制。

### 2.6 结局系统

- 第四章最后一节点结束后，根据好感度与贴合度判定结局类型：

| 结局       | 好感度 | 贴合度 | 说明     |
|------------|--------|--------|----------|
| 灵魂伴侣   | ≥80    | ≥70    | 双高契合 |
| 表面和谐   | ≥80    | <50    | 讨好对方、失去自我 |
| 知己难求   | <60    | ≥70    | 三观合但不来电 |
| 分道扬镳   | <50    | <50    | 完全不合 |

- 其余组合由规则就近归类（见实现中的 `getEndingType`）。
- 结局页展示对应标题、副标题、评语，并调用「AI 评语」模块（MVP 为模拟）。

### 2.7 AI 评语（MVP：模拟版）

- 结局页展示一段 2～4 句的评语。
- MVP 阶段：根据结局类型、好感度、贴合度等用规则/模板生成，不调用外部 AI 接口。
- 评语可写入玩家档案的 `endingHistory`，供心域地图/个人档案查看。

### 2.8 玩家档案与心域地图

- **本地存储**：`PlayerProfile` 含 `unlockedScripts`、`characterCollection`、`endingHistory`、`badges` 等，持久化到 `localStorage`。
- **结局时**：将本次结果（时间、剧本 ID、对方 MBTI、结局类型、好感度、贴合度、AI 评语）追加到 `endingHistory`，并更新 `characterCollection` 对应 MBTI 的攻略次数、最佳数值等。
- **心域地图**：展示 MBTI 角色图鉴（攻略次数、达成结局）、可含简单勋章（如「首次灵魂伴侣」）。历史记录列表可展示最近若干条含 AI 评语。
- **个人档案**：展示昵称、魅力等级、已解锁剧本等基础信息（MVP 可精简为只读展示）。

## 3. 功能需求详情（MVP 范围）

### 3.1 数据与存储

- **剧本**：`public/data/scripts/` 下每剧本一个 JSON（如 `modern_love.json`），包含 `id`、`title`、`description`、`recommendedMbti`、`characters`、`nodes`。列表由 `index.json` 提供。
- **玩家档案**：`localStorage` 键 `storyflow_profile`，结构见类型定义（`PlayerProfile`、`EndingRecord`、`CharacterRecord`）。
- **音频设置**：`localStorage` 键 `storyflow_audio_settings`，保存 BGM/音效/环境音音量及静音开关。

### 3.2 剧本引擎（已实现）

- 支持效果：`affection`（固定或 valueByMbti）、`alignment`（固定或 valueByPlayerMbti）、`jobEffect`、`setFlag`。
- 支持条件：`affection`、`alignment`、`playerMbti`、`playerJob`、`flag`。
- 节点可带 `scene` 字段，用于场景 BGM 切换（如咖啡店、公园、餐厅）。

### 3.3 音频（MVP 必做）

- **P0**：主菜单 BGM、剧情场景 BGM（按节点 scene 切换）、按钮点击音效、好感度/贴合度变化音效。
- **音量**：设置面板提供 BGM/音效（及可选环境音）滑块与静音，并持久化。
- 实现：Howler.js 封装为 AudioManager，预加载、播放、停止、淡入淡出；首次播放需在用户手势后触发（避免自动播放被拦截）。

### 3.4 移动端与部署（MVP 必做）

- 主要页面具备基础响应式（字体、按钮触控区域、容器宽度），在手机浏览器可读可点。
- 生产构建后部署至静态托管（如 Vercel），配置 SPA 回退（所有路径回退到 `index.html`）。

## 4. MVP 边界（本周五上线）

### 4.1 本周必须完成

- 剧本大厅 → 创建角色 → 选对象 → 四章剧情（双数值）→ 四型结局 → 模拟 AI 评语 → 写入档案。
- 心域地图可查看角色图鉴与历史记录（含评语）。
- 主菜单 + 剧情场景 BGM、基础 UI 音效、音量设置。
- 移动端基本适配、全流程无阻塞、部署上线可访问。

### 4.2 可延后

- 真实 AI 接口接入。
- 多套独立剧本内容（当前可 1～2 套剧本，其中一套可为占位或复用节点）。
- 勋章墙丰富化、攻略地图树状图、分享卡片等。

## 6. 风险与应对

| 风险               | 应对 |
|--------------------|------|
| 音频自动播放被拦截 | 所有 BGM/音效在用户点击「开始游戏」或进入剧本后再初始化/播放 |
| 部署后刷新 404     | 配置 `vercel.json` 的 rewrites 将 `/(.*)` 指向 `/index.html` |
| 剧本数据未就绪     | 至少保证一个剧本（如都市迷情）节点完整可通关；第二剧本可复用或占位 |
