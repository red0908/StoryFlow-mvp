# 姓名库、职业列表、MBTI 列表 — 放置位置与格式说明

根据**简化版执行计划**与**需求文档**，游戏需要「随机生成 5 位异性候选人」，候选人数据来自姓名库、职业列表和 MBTI 列表。以下说明这些数据的**放置位置**、**文件格式**及**示例**。

---

## 一、放置位置

与剧本 `story.json` 同级，放在前端项目的 **`public/data/`** 目录下：

```
StoryFlow-mvp-code/
  public/
    data/
      story.json          # 剧本节点（已有）
      candidates.json     # 可选：预置候选人示例（已有）
      gameConfig.json     # 姓名库 + 职业列表 + MBTI 列表（新建）
```

- **原因**：需求文档 4.1 约定「所有数据（剧本、角色库）打包在静态资源中」；执行计划中用户任务是在本地准备 `story.json` 与姓名库等，与剧本同属「配置数据」，放在 `public/data/` 便于前端通过 `/data/gameConfig.json` 请求加载，且与现有 `story.json`、`candidates.json` 一致。

---

## 二、文件格式与用途

| 内容 | 用途 | 在 gameConfig 中的键 |
|------|------|----------------------|
| 姓名库（男/女各 5 个） | 随机生成候选人时，按玩家性别从异性姓名池中抽取 | `names.male` / `names.female` |
| 职业列表（5 种） | 角色创建时 5 选 1、候选人生成时随机分配 | `jobs` |
| MBTI 列表（4 种） | 角色创建时 4 选 1、候选人生成时随机分配 | `mbti` |

- **执行计划**：姓名男/女各 5 个，职业 5 种（医生、教师、工程师、设计师、律师），MBTI 四种（ENFJ/INTJ/INFJ/ENTJ）。
- **需求文档**：候选人含姓名、年龄、职业、MBTI、一句话介绍；职业与 MBTI 与上述一致。`gameConfig.json` 提供「池子」，代码中用随机数从中抽取并生成 `Candidate`。

---

## 三、gameConfig.json 示例

以下为可直接使用的 **`public/data/gameConfig.json`** 示例（满足执行计划与需求文档）：

```json
{
  "names": {
    "male": ["陈浩", "李想", "王磊", "张俊", "刘洋"],
    "female": ["林然", "周舟", "吴雨", "郑悦", "孙琳"]
  },
  "jobs": [
    { "id": "doctor", "label": "医生" },
    { "id": "teacher", "label": "教师" },
    { "id": "engineer", "label": "工程师" },
    { "id": "designer", "label": "设计师" },
    { "id": "lawyer", "label": "律师" }
  ],
  "mbti": [
    { "id": "ENFJ", "label": "ENFJ", "tagline": "热情的教育者" },
    { "id": "INTJ", "label": "INTJ", "tagline": "理性的战略家" },
    { "id": "INFJ", "label": "INFJ", "tagline": "安静的洞察者" },
    { "id": "ENTJ", "label": "ENTJ", "tagline": "果断的领导者" }
  ]
}
```

- **names**：`male` / `female` 各 5 个，用于按玩家性别生成异性候选人姓名。
- **jobs**：5 种职业；`id` 与需求文档里 `Player.job`、`Candidate.job` 的 string 一致，`label` 用于界面展示。
- **mbti**：4 种人格；`id` 与类型 `'ENFJ'|'INTJ'|'INFJ'|'ENTJ'` 一致，`tagline` 可用于「一句话介绍」生成规则（如执行计划中的 `{name}，{age}岁，{job}，{mbti}性格` 或与 `tagline` 组合）。

---

## 四、与候选人「一句话介绍」的配合

执行计划第二天用户任务：  
「为候选人一句话介绍编写简单生成规则（如 `{name}，{age}岁，{job}，{mbti}性格`）。」

- **job**：可从 `gameConfig.jobs` 中用 `id` 取对应 `label` 填入介绍。
- **mbti**：可从 `gameConfig.mbti` 中用 `id` 取对应 `tagline` 或仅用 `label`。
- 示例：`"{name}，{age}岁，{jobLabel}，{mbtiTagline}"` → 「林然，26岁，设计师，热情的教育者」。

---

## 五、小结

| 项目 | 说明 |
|------|------|
| **放置位置** | `public/data/gameConfig.json`（与 story.json 同级） |
| **格式** | 单个 JSON 文件，包含 `names`（male/female 数组）、`jobs`（id+label）、`mbti`（id+label+tagline） |
| **数量** | 男名 5、女名 5；职业 5；MBTI 4，与简化版执行计划一致 |

将上述示例保存为 **`Code/StoryFlow-mvp-code/public/data/gameConfig.json`** 即可在实现「候选人生成界面」时通过请求 `/data/gameConfig.json` 读取并用于随机生成。
