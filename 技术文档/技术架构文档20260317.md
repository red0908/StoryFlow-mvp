## 2. 项目概述与技术选型

### 2.1 项目定位

- **产品形态**：MBTI 主题的轻量级相亲互动游戏，单机、纯前端。
- **核心流程**：首页 → 剧本大厅（选剧本）→ 创建角色 → 选择相亲对象（5 选 1）→ 四章剧情（选项驱动，好感度+贴合度）→ 四型结局 → 模拟 AI 评语 → 更新玩家档案；可进入心域地图/个人档案查看收集与历史。
- **技术原则**：纯前端、无后端；剧本与配置由 JSON 驱动；玩家档案与音频设置持久化于 localStorage。

### 2.2 技术选型总览

| 类别         | 选型              | 说明 |
|--------------|-------------------|------|
| 运行时       | 浏览器            | 现代浏览器，支持 ES Module |
| 框架         | React             | ^18.2.0 |
| 语言         | TypeScript        | ^5.x |
| 构建工具     | Vite              | ^4.x |
| 路由         | react-router-dom  | ^7.x |
| 状态管理     | Zustand           | ^4.x（usePlayerStore / useGameStore / useScriptStore / useProfileStore） |
| 样式         | Less + Tailwind CSS | 页面级 Less，工具类 Tailwind |
| 音频         | Howler.js         | BGM / 环境音 / UI 音效，封装为 audioManager |
| 数据         | 静态 JSON + localStorage | 剧本：`/data/scripts/`；配置：`gameConfig.json`；档案：`storyflow_profile`；音频设置：`storyflow_audio_settings` |
| 部署         | 静态托管          | 推荐 Vercel，需 SPA 回退 |

## 3. 整体架构

### 3.1 架构形态

- **单体 SPA**：单入口 `index.html` → `main.tsx` → `App.tsx`，所有页面通过 React Router 切换。
- **无后端**：数据来自静态 JSON 或内存状态；AI 评语 MVP 阶段为前端模拟。
- **状态与数据流**：
  - **全局状态**：Zustand 管理「玩家」「游戏进行」「当前剧本」「玩家档案」。
  - **页面级状态**：各页面 useState/useEffect 管理本地 UI、剧本加载等。
  - **数据源**：`/data/scripts/index.json`（剧本列表）、`/data/scripts/<id>.json` 或 `/data/story.json`（剧本节点）、`/data/gameConfig.json`（职业、MBTI、姓名库）。

### 3.2 核心数据流简图

```
[ 用户操作 ]
     ↓
[ 页面组件 ] ←→ [ Zustand：usePlayerStore / useGameStore / useScriptStore / useProfileStore ]
     ↓                    ↓
[ 路由跳转 ]         [ 剧本 JSON / gameConfig / localStorage ]
     ↓                    ↓
[ 下一页面 ]         [ 剧情引擎：节点、选项、双数值、效果/条件 ]
     ↓
[ 音频管理器 ] ← 场景/BGM/SFX 触发
```

- **剧本大厅**：读取 `index.json`，点击剧本调用 `useScriptStore.setCurrentScript`，跳转创建角色。
- **创建角色**：表单写入 `usePlayerStore`；可选 localStorage 存最近角色（键如 `storyflow_player`）。
- **选对象**：根据 gameConfig + 剧本 characters 生成 5 位候选人，选择后 `useGameStore.startGame(opponent)`，跳转 `/game`。
- **剧情页**：按 `useScriptStore.currentScriptId` 加载对应剧本 JSON，根据 `currentNodeId` 渲染节点与选项；选项点击后执行效果（好感/贴合/职业/flag）、跳转 nextNode，无下一节点时进入 `/ending`。
- **结局页**：`getEndingType(affection, alignment)` 得到四型之一，展示配置的标题/副标题/评语；调用模拟 AI 评语并写入 `useProfileStore.appendEnding`、`bumpEncounter`；可跳转心域地图或首页。
- **心域地图 / 个人档案**：读取 `useProfileStore.profile`，展示 characterCollection、endingHistory、badges 等。

## 4. 技术栈详情

### 4.1 开发与构建

- **Vite**：`vite.config.ts` 使用 `@vitejs/plugin-react`，开发服务器默认可配置 `open: true`。
- **TypeScript**：根目录 `tsconfig` 约束；`src/vite-env.d.ts` 声明 Vite 客户端类型及 `*.less` 模块类型。
- **入口**：`index.html` 挂载 `#root`，脚本 `src/main.tsx`；`main.tsx` 引入全局 `index.less` 后渲染 `<App />`。

### 4.2 样式体系

- **Less**：全局 `src/index.less` 引入 Tailwind 三件套，设基础字体与 `#root` 全高。
- **Tailwind**：`tailwind.config.js` 配置 `content` 扫描路径；页面内结合类名与 Less 使用。
- **页面样式**：各页面目录下同名 `.less`（如 `GamePage.less`、`EndingPage.less`）。

### 4.3 字体与静态资源

- **字体**：`index.html` 中通过 Google Fonts 等引入 Noto Serif SC、ZCOOL XiaoWei 等。
- **静态资源**：放在 `public/`，引用以 `/` 开头，如 `/background/`、`/person_img/`、`/data/`、`/audio/`。

## 7. 状态管理（Zustand）

### 7.1 usePlayerStore

- **状态**：`player: Player | null`（性别、年龄、职业、MBTI、一句话介绍等）。
- **方法**：`setPlayer`、`resetPlayer`。
- **使用处**：CreateRole 写入；BlindDatePage、GamePage、EndingPage 读取。

### 7.2 useGameStore

- **状态**：`opponent`、`currentChapter`、`currentNodeId`、`affection`（0–100）、`alignment`（0–100）、`flags`。
- **方法**：`setOpponent`、`setCurrentNode`、`addAffection`、`setAffection`、`addAlignment`、`setAlignment`、`setFlag`、`resetGame`、`startGame(opponent)`。
- **使用处**：BlindDatePage 调用 `startGame`；GamePage 读写节点与双数值；EndingPage 读取 opponent/affection/alignment 并调用 `resetGame`。

### 7.3 useScriptStore

- **状态**：`currentScriptId: string | null`、`currentScript: ScriptSummary | null`。
- **方法**：`setCurrentScript(script)`、`resetScript()`。
- **使用处**：ScriptHall 选择剧本后 setCurrentScript；CreateRole/BlindDatePage/GamePage 读取 currentScriptId 用于候选人生成与剧本加载。

### 7.4 useProfileStore

- **状态**：`profile: PlayerProfile`（displayName、charmLevel、charmExp、unlockedScripts、characterCollection、endingHistory、badges），从 localStorage 初始化。
- **方法**：`setDisplayName`、`unlockScript`、`appendEnding(record)`、`bumpEncounter(mbti, patch)`。
- **使用处**：EndingPage 结局时 appendEnding、bumpEncounter；HeartMapPage、ProfilePage 读取 profile 展示。

## 8. 类型定义（types.ts）要点

- **MBTI**：`'ENFJ'|'INTJ'|'INFJ'|'ENTJ'`。
- **EndingType**：`'success'|'surface'|'friend'|'fail'`。
- **Player**：性别、年龄、职业、MBTI、一句话介绍等。
- **Candidate**：id、姓名、年龄、职业、MBTI、tagline。
- **GameState**：opponent、currentChapter、currentNodeId、affection、alignment、flags。
- **Effect**：`affection`（value / valueByMbti）、`alignment`（value / valueByPlayerMbti）、`setFlag`、`jobEffect`。
- **Condition**：`affection`/`alignment`（operator + value）、`playerMbti`、`playerJob`、`flag`。
- **StoryNode**：id、chapter、scene（可选）、text、options。
- **Script**：id、title、description、recommendedMbti、characters、nodes。
- **PlayerProfile**：displayName、charmLevel、charmExp、unlockedScripts、characterCollection、endingHistory、badges。
- **EndingRecord**：date、scriptId、opponentMbti、endingType、affection、alignment、aiComment。

## 9. 剧情引擎逻辑（GamePage）

- **剧本加载**：根据 `useScriptStore.currentScriptId` 决定请求 `/data/story.json` 或 `/data/scripts/${currentScriptId}.json`；返回 Script 时取 `data.nodes`，否则按 StoryNode[] 处理。
- **当前节点**：由 `useGameStore.currentNodeId` 在 nodes 中查找；若为空且有数据，则用当前章节第一个节点初始化。
- **文本插值**：`interpolateText(text, { opponent, player })` 替换 `{opponent.name}`、`{opponent.mbti}`、`{player.description}` 等。
- **选项条件**：`evaluateSingleCondition` 支持 affection、alignment、playerMbti、playerJob、flag；`filterOptions` 得到可见选项。
- **选项效果**：`applyEffects(effects, ctx)` 支持 affection（value/valueByMbti）、alignment（value/valueByPlayerMbti）、setFlag、jobEffect（按玩家职业加成）；同时可调用 `computeEffectDeltas` 用于播放好感/贴合变化音效。
- **跳转**：选项的 `nextNode` 存在则 `setCurrentNode` 并同步 URL chapter；不存在则 `navigate('/ending')`。
- **音频**：根据当前节点 `scene` 或章节（CHAPTER_BGM/CHAPTER_AMBIENT）切换 BGM/环境音；选项点击与数值变化时播放对应 SFX（通过 audioManager）。

---

## 10. 结局与档案（EndingPage）

- **结局判定**：`getEndingType(affection, alignment, urlType)` 返回四型之一；若 URL 带 `type=success|surface|friend|fail` 可覆盖。
- **展示**：从 `ENDINGS[endingType]` 取 title、subtitle、comment；comment 做 `{opponent.name}` 等插值。
- **模拟 AI 评语**：MVP 使用 `generateMockAIComment(endingType, affection, alignment, opponent)` 生成 2～4 句模板评语，展示并写入 `EndingRecord.aiComment`。
- **档案更新**：`appendEnding({ date, scriptId, opponentMbti, endingType, affection, alignment, aiComment })`；`bumpEncounter(opponent.mbti, { soulmate, endingId, affection, alignment })`。
- **音频**：结局页可根据结局类型播放对应 BGM（如 ending_perfect、ending_regret）。

---

## 11. 音频（audioManager）

- **封装**：Howler.js，单例 `audioManager`，提供 `preload()`、`playBGM(id, fadeIn)`、`stopBGM(fadeOut)`、`playAmbient(id)`、`playSFX(id)`、`setVolume(type, value)`、`setMuted(muted)`、`stopAll()` 等。
- **常量**：`src/audio/constants.ts` 定义 BGMId（main_menu、coffee、park、restaurant、ending_*）、AmbientId、SFXId（ui_click、ui_affection_up/down、ui_alignment 等）及路径；`CHAPTER_BGM`、`CHAPTER_AMBIENT` 映射章节与 BGM/环境音。
- **持久化**：`storyflow_audio_settings` 存 bgmVolume、sfxVolume、ambientVolume、muted。
- **策略**：首次播放需在用户手势后（如点击「开始游戏」或进入剧本），避免自动播放被拦截。

## 12. 数据与配置

### 12.1 /data/scripts/index.json

剧本列表，每项含 `id`、`title`、`description`、`recommendedMbti`、`cover` 等，ScriptHall 请求该文件渲染卡片。

### 12.2 剧本 JSON（/data/scripts/<id>.json 或 /data/story.json）

- **格式**：`Script` 或直接 `StoryNode[]`。Script 含 `id`、`title`、`description`、`recommendedMbti`、`characters`、`nodes`。
- **节点**：id、chapter、scene（可选）、text、options。option 含 text、nextNode、effects、condition（可选）。

### 12.3 gameConfig.json

names（male/female）、jobs、mbti，供创建角色与 generateCandidates 使用。

## 13. 构建与部署

- **开发**：`npm run dev`。
- **构建**：`npm run build`，产出 `dist/`。
- **预览**：`npm run preview`。
- **部署**：将 `dist/` 部署至静态托管；若使用 Vercel，根目录添加 `vercel.json`：
  ```json
  { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
  ```
  避免 SPA 刷新非根路径 404。
