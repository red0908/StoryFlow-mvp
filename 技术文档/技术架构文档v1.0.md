## 2. 项目概述与技术选型

### 2.1 项目定位

- **产品形态**：MBTI 主题的轻量级相亲互动游戏（单机、纯前端）。
- **核心流程**：首页 → 创建角色 → 选择相亲对象（5 选 1）→ 四章剧情（选项驱动）→ 结局（按好感度分三种）。
- **技术原则**：纯前端、无后端；剧本与配置由 JSON 驱动；无本地持久化游戏进度（每次新游戏可刷新状态）。

### 2.2 技术选型总览

| 类别         | 选型              | 版本/说明 |
|--------------|-------------------|-----------|
| 运行时       | 浏览器            | 现代浏览器，支持 ES Module |
| 框架         | React             | ^18.2.0 |
| 语言         | TypeScript        | ^5.0.0 |
| 构建工具     | Vite              | ^4.0.0 |
| 路由         | react-router-dom  | ^7.13.1 |
| 状态管理     | Zustand           | ^4.5.0 |
| 样式         | Less + Tailwind CSS | Less ^4.6.4，Tailwind ^3.0.0 |
| 数据驱动     | 静态 JSON         | `public/data/story.json`、`gameConfig.json` |

与需求文档中的「React + Zustand + Tailwind、剧本 JSON 驱动」一致；实际样式采用 **Less** 做页面级样式，**Tailwind** 做工具类，与设计文档中的视觉规范配合使用。

## 3. 整体架构

### 3.1 架构形态

- **单体 SPA**：单入口 `index.html` → `main.tsx` → `App.tsx`，所有页面在同一应用内通过 React Router 切换。
- **无后端**：所有数据来自前端静态资源（JSON）或内存状态，不涉及服务端接口。
- **状态与数据流**：
  - **全局状态**：Zustand 管理「玩家」与「游戏进行状态」。
  - **页面级状态**：各页面内部 useState/useEffect 管理本地 UI、加载配置与剧本。
  - **数据源**：`/data/gameConfig.json`（职业、MBTI、姓名库）、`/data/story.json`（剧本节点）。

### 3.2 核心数据流简图

```
[ 用户操作 ]
     ↓
[ 页面组件 ] ←→ [ Zustand Store：usePlayerStore / useGameStore ]
     ↓                    ↓
[ 路由跳转 ]         [ 剧本/配置 JSON ]
     ↓                    ↓
[ 下一页面 ]         [ 剧情引擎：节点、选项、好感度 ]
```

- **创建角色**：表单数据写入 `usePlayerStore`，可选写入 `localStorage`（仅角色信息，键：`storyflow_player`）。
- **选择对象**：根据 `gameConfig` + 玩家性别生成 5 位候选人，选择后调用 `useGameStore.startGame(opponent)`，跳转 `/game`。
- **剧情页**：从 `story.json` 加载节点列表，根据 `currentNodeId` 渲染当前节点文本与选项；选项点击后执行好感效果、跳转 `nextNode`，无下一节点时进入 `/ending`。
- **结局页**：根据 `affection`（及可选 URL `type`）决定三种结局之一，展示评语与「重新开始」；重新开始会 `resetGame()` 并回到首页。

## 4. 技术栈详情

### 4.1 开发与构建

- **Vite**：`vite.config.ts` 使用 `@vitejs/plugin-react`，开发服务器默认 `open: true`。
- **TypeScript**：根目录 `tsconfig` 约束；`src/vite-env.d.ts` 声明 Vite 客户端类型及 `*.less` 模块类型。
- **入口**：`index.html` 挂载 `#root`，脚本为 `src/main.tsx`；`main.tsx` 引入全局 `index.less` 后渲染 `<App />`。

### 4.2 样式体系

- **Less**：全局 `src/index.less` 引入 Tailwind 三件套（base/components/utilities），并设基础字体与 `#root` 全高。
- **Tailwind**：`tailwind.config.js` 仅配置 `content` 扫描路径（`index.html`、`src/**/*.{js,ts,jsx,tsx}`），主题为默认扩展。
- **PostCSS**：`postcss.config.cjs` 使用 `tailwindcss`、`autoprefixer`。
- **页面样式**：各页面目录下均有同名 `.less` 文件（如 `HomePage.less`、`GamePage.less`），采用类名 + 部分 Tailwind 工具类，与设计文档中的视觉规范对应。

### 4.3 字体与静态资源

- **字体**：`index.html` 中通过 Google Fonts 引入 Noto Serif SC、Cormorant Garamond、ZCOOL XiaoWei 等。
- **静态资源**：放在 `public/`，引用路径以 `/` 开头，如 `/background/bg-01.png`、`/person_img/avatar_*.png`、`/data/story.json`、`/data/gameConfig.json`。

## 6. 核心模块说明

### 6.1 路由与页面（App.tsx）

| 路径               | 组件          | 说明 |
|--------------------|---------------|------|
| `/`                | HomePage      | 首页：玩法介绍、开始游戏 → 创建角色 |
| `/create/myRole`   | CreateRole    | 创建角色：性别、年龄、职业、MBTI、一句话介绍 |
| `/blindDate`       | BlindDatePage | 选择相亲对象：5 位候选人，确认后 startGame → /game |
| `/game`            | GamePage      | 剧情主界面：章节、节点、选项、好感度 |
| `/ending`          | EndingPage    | 结局：按好感度/URL 显示三种结局之一，重新开始 |
| `/select`          | SelectPage   | 占位，未接入主流程 |

路由由 `react-router-dom` 的 `BrowserRouter` + `Routes` + `Route` 实现，无嵌套路由。

### 6.2 状态管理（Zustand）

**usePlayerStore**

- **状态**：`player: Player | null`（性别、年龄、职业、MBTI、一句话介绍）。
- **方法**：`setPlayer`、`resetPlayer`。
- **使用处**：CreateRole 写入；BlindDatePage、GamePage、EndingPage 读取展示或校验。

**useGameStore**

- **状态**：`opponent`、`currentChapter`、`currentNodeId`、`affection`（0–100，初始 50）、`flags`。
- **方法**：`setOpponent`、`setCurrentNode`、`addAffection`、`setAffection`、`setFlag`、`resetGame`、`startGame(opponent)`。
- **使用处**：BlindDatePage 调用 `startGame`；GamePage 读写当前节点与好感度；EndingPage 读取 opponent/affection 并调用 `resetGame`。

### 6.3 类型定义（types.ts）

- **MBTI**：字面量联合 `'ENFJ'|'INTJ'|'INFJ'|'ENTJ'`。
- **Player**：性别、年龄、职业 id、MBTI、一句话介绍。
- **Candidate**：id、姓名、年龄、职业 id、MBTI、一句话介绍（tagline）。
- **GameState**：对手、当前章节、当前节点 id、好感度、flags。
- **剧本相关**：`AffectionEffect`、`AffectionCondition`、`StoryOption`、`StoryNode`（id、chapter、text、options）。

与 `story.json`、`gameConfig.json` 及需求文档中的数据结构一致。

### 6.4 剧情引擎逻辑（GamePage 内）

- **剧本加载**：页面挂载时 `fetch('/data/story.json')`，得到 `StoryNode[]`。
- **当前节点**：由 `useGameStore.currentNodeId` 在 nodes 中查找；若为空且有节点数据，则用当前章节第一个节点初始化。
- **文本插值**：`interpolateText(text, { opponent, player })` 替换 `{opponent.name}`、`{opponent.mbti}`、`{player.description}` 等。
- **选项条件**：`evaluateCondition(condition, affection)` 支持 `<=`、`>=`、`<`、`>`、`===`；`filterOptions(options, affection)` 得到可见选项。
- **选项效果**：`applyEffects(effects, opponentMbti, addAffection)` 支持固定 `value` 或按 MBTI 的 `valueByMbti` 增减好感度。
- **跳转**：选项的 `nextNode` 在 nodes 中存在则 `setCurrentNode` 并更新 URL `chapter`；不存在则 `navigate('/ending')`。

### 6.5 工具与数据

- **generateCandidates(config, playerGender)**：按玩家性别从 `config.names` 取异性姓名池，随机抽 5 个名字，再为每人随机年龄（25–35）、职业、MBTI，生成 tagline 与唯一 id，返回 `Candidate[]`。
- **getJobLabel(config, jobId)**：根据 `gameConfig.jobs` 返回职业中文标签。
- **gameConfig 扩展**：`generateCandidates.ts` 中定义 `GameConfigFull`（含 `names.male/female`），与 `gameConfig.json` 结构一致。

## 7. 数据与配置

### 7.1 gameConfig.json

- **names**：`male`、`female` 姓名数组，供候选人随机取名。
- **jobs**：`{ id, label }[]`，职业列表，创建角色与候选人共用。
- **mbti**：`{ id, label, tagline }[]`，MBTI 列表，与 `types.MBTI` 一致。

创建角色页与 BlindDate 页通过 `fetch('/data/gameConfig.json')` 加载，用于表单项与候选人生成。

### 7.2 story.json

- **格式**：`StoryNode[]`。
- **节点字段**：`id`、`chapter`（1–4）、`text`（可含 `{opponent.name}` 等占位）、`options`。
- **选项字段**：`text`、`nextNode`、`effects`（affection 的 add/subtract，支持 value 或 valueByMbti）、可选 `condition`（affection 比较）。

当前示例为单节点；扩展时按章节增加节点并串好 `nextNode`，最后一节无下一节点时由 GamePage 跳转结局。

### 7.3 结局判定（EndingPage）

- **类型**：`success`（灵魂伴侣）、`regret`（略有遗憾）、`fail`（分道扬镳）。
- **规则**：优先读 URL `type=success|regret|fail`；否则按 `affection`：≥80 → success，≥50 → regret，<50 → fail。
- **展示**：标题、副标题、评语（评语中可插值 `{opponent.name}`）、对方头像/姓名/MBTI、最终好感度、重新开始按钮。

## 8. 构建与运行

### 8.1 脚本

- **开发**：`npm run dev`（Vite 开发服务器，默认打开浏览器）。
- **构建**：`npm run build`（产出到 `dist/`）。
- **预览构建结果**：`npm run preview`。

### 8.2 部署

- 静态托管即可：将 `dist/` 内容部署至任意静态服务器或 CDN；若使用 Vercel/Netlify 等，可配置根目录为代码仓库、构建命令 `npm run build`、输出目录 `dist`。
- 路由为 Browser History：部署时需配置 SPA 回退（所有路径回退到 `index.html`），否则刷新非根路径会 404。

## 9. 扩展与约束

### 9.1 当前约束

- **无后端、无用户系统**：不涉及登录、存档同步。
- **无持久化进度**：游戏状态仅内存 + 可选 localStorage 存「创建角色」信息；刷新或重新开始会丢失当前局进度。
- **剧本规模**：节点数、章节数由 `story.json` 决定，引擎支持多章节、多节点与条件选项。
- **分辨率**：部分页面使用 `min-w-[1280px]`，当前以 PC 端为主；移动端可后续做响应式与触控优化。

### 9.2 可扩展点

- **剧本**：在 `story.json` 中增加节点、完善 `nextNode` 与 `effects`/`condition`，即可扩展分支与好感逻辑。
- **配置**：扩展 `gameConfig.json`（如新职业、新 MBTI）需同步 `types` 与 `generateCandidates` 等使用处。
- **结局**：在 EndingPage 中扩展 `ENDINGS` 或结局判定规则即可增加新结局类型。
- **状态**：若需更多全局状态，可在 `stores/` 下新增 Zustand store 或在现有 store 上扩展字段。
