## 1. 文档说明

### 1.1 目的

本文件在**已跑通并准备上线的 StoryFlow MVP**（相亲互动叙事游戏）基础上，承接早期「心域奇旅」**游戏大平台**设想，将二者统一为同一产品愿景下的分层需求：明确**当前已交付能力**、**平台化扩展模块**与**阶段性目标**，供研发、运营、合作与融资对齐。

### 1.2 文档关系

| 文档 | 说明 |
|------|------|
| 《需求文档20260317》等 | MVP 单游戏定稿范围与验收 |
| 《完整设计文档》 | MVP 技术、路由、移动端与部署 |
| 《官网设计》 | 品牌、官网信息架构与生态预告 |
| **本文档** | **大平台级**愿景、模块地图、阶段规划与非功能要求 |

### 1.3 术语

| 术语 | 含义 |
|------|------|
| **叙事核心（Story Core）** | 当前 MVP：多剧本、双数值、四结局、心域地图与档案的完整闭环 |

## 2. 愿景与定位

### 2.1 一句话愿景

打造一个**安全、有趣、有温度**的**数字化心灵游乐场** —— 融合**互动叙事、轻游戏化体验与专业心理引导入口**，让用户在故事里遇见自己、理解关系、并知道何时寻求现实支持。

### 2.2 与 MVP 的关系

- **MVP 不是「小 demo」**，而是平台的 **v0 内核**：已验证「创建角色 → 沉浸式分支叙事 → 双维度反馈 → 结局与收集」的可玩性与可扩展剧本架构。
- **大平台** = 在叙事核心之上，逐步叠加 **账号与云同步、UGC/角色工坊、社区、心境导航、多类型剧本与商业化**，形成可持续生态。

### 2.3 差异化要点

| 维度 | 平台级主张 |
|------|------------|
| 人格与叙事 | MBTI 等框架服务于**剧情与反思**，不做刻板标签定论 |
| 玩法 | **好感度 + 贴合度** 双轨，强调关系与自我认同的平衡（已 MVP 实现） |
| 体验 | 场景化音频、图鉴与档案，强化**仪式感与回顾**（已 MVP 实现） |
| 边界 | 游戏化自我探索 + **心境模块**引导深度需求至专业服务，明确非治疗承诺 |

## 3. 平台架构总览

### 3.1 逻辑分层

```
┌─────────────────────────────────────────────────────────────┐
│  品牌与获客层：官网、应用商店、活动与投放                      │
├─────────────────────────────────────────────────────────────┤
│  平台壳层（演进中）：账号、支付、合规、推荐、消息              │
├─────────────────────────────────────────────────────────────┤
│  叙事核心 Story Core（MVP 已上线）                           │
│  剧本大厅 · 角色创建 · 选人 · 章节剧情 · 四结局 · 模拟评语     │
│  心域地图 · 个人档案 · 音频与设置                             │
├─────────────────────────────────────────────────────────────┤
│  扩展模块（分期）                                            │
│  角色工坊 · 交流社区 · 心境 · 拯救者/探索者剧本线 · 组队/共创  │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 模块地图

| 模块 | 状态 | 平台级职责 |
|------|------|------------|
| **叙事核心** | **已交付（MVP）** | 多剧本入口、剧本引擎、双数值、结局、档案、心域地图、沉浸音频 |
| **品牌官网** | 规划中 | 愿景传达、试玩导流、生态预告（见《官网设计》） |
| **账号与云档案** | 未开始 | 跨端同步结局历史、图鉴、设置；防丢失与社交基础 |
| **真实 AI 评语 / 对话** | 未开始 | 在合规与成本可控下增强反馈与个性化（替代或增强模拟评语） |
| **角色工坊** | 未开始 | UGC：自定义角色 IP、MBTI/信念/背景；与审核、分账联动 |
| **交流社区** | 未开始 | 攻略、同人、话题、轻社交；与举报、未成年人策略联动 |
| **心境** | 未开始 | 心理科普、自助工具索引、**正规渠道心理咨询/援助入口**（平台仅导航与撮合，服务由持证方提供） |
| **拯救者 / 探索者剧本** | 未开始 | 多元剧本类型（如成长、职场、家庭）；探索者线 + 引导者/咨询师共创内容 |

## 4. 用户与场景

### 4.1 用户画像

| 角色 | 诉求 | 平台演进中的触点 |
|------|------|------------------|
| **探索者（玩家）** | 短平快情感体验、MBTI 趣味、收集与复盘 | 叙事核心 → 社区分享 → 更多剧本 |
| **创作者 / 引导者** | 表达、副业、专业延伸 | 角色工坊、官方共创计划、心境合作方 |
| **心理服务使用者** | 情绪困扰需真人帮助 | 心境：自助 + 转介专业资源 |
| **合作伙伴** | 内容、渠道、EAP、教育 | B2B 剧本包、主题活动、数据洞察（脱敏聚合） |

### 4.2 核心用户旅程（与 MVP 一致，可扩展）

1. 进入产品 → 剧本大厅选本 → 创建角色 → 选择对象 → 四章互动 → 四型结局之一 → 模拟 AI 评语 → 写入档案 → 心域地图 / 个人档案回顾。  
2. **平台期**：同一用户可登录、在多设备继续、在社区发帖、在心境预约或获取热线信息。

## 5. 功能需求分层

### 5.1 L0 — 叙事核心（当前 MVP，大平台的「最小内核」）

**范围与验收口径与《需求文档20260317》对齐，摘要如下：**

| 能力 | 要求 |
|------|------|
| 剧本大厅 | `index.json` 驱动多剧本选择与 scriptId 贯通全流程 |
| 角色创建 | 性别、年龄、职业、MBTI（四型）、一句话介绍 |
| 选人 | 每局 5 选 1 异性候选人（剧本绑定） |
| 剧情 | 节点插值、选项 effects（好感/贴合/职业/flag）、条件可见性、场景 BGM |
| 结局 | 四型：灵魂伴侣 / 表面和谐 / 知己难求 / 分道扬镳 |
| 评语 | MVP 规则模板生成 2～4 句，写入 `endingHistory` |
| 档案与地图 | localStorage 持久化；心域地图图鉴与历史；个人档案与音量设置 |

**平台文档约束**：L0 的剧本 Schema、结局类型、档案结构视为**平台内容协议 v0**，后续云同步与 UGC 需向后兼容或提供迁移策略。

### 5.2 L1 — 平台壳与增长（上线后 0～6 个月，优先级由高到低）

| 需求 ID | 需求描述 | 备注 |
|---------|----------|------|
| P-101 | 用户账号体系（手机/邮箱/第三方可选） | 与合规、未成年人模式绑定 |
| P-102 | 档案与设置云同步 | 替代或并存 localStorage |
| P-103 | 官网与游戏同品牌域名导流 | 见《官网设计》 |
| P-104 | 真实 AI 评语（可选开关） | 提示词安全、不输出诊断 |
| P-105 | 第二套完整独立剧本上线 | 验证多本运营与数据隔离 |
| P-106 | 分享卡片 / 结局海报（可选） | 拉新与社交传播 |

### 5.3 L2 — 创作与社区（6～18 个月）

| 需求 ID | 需求描述 | 备注 |
|---------|----------|------|
| P-201 | **角色工坊**：创建训练式角色卡（MBTI、信念、口癖、立绘槽位） | 审核流程、版权与真人肖像合规 |
| P-202 | **交流社区**：帖子、话题、轻互动 | 举报、审核、青少年保护 |
| P-203 | 官方「共创剧本」招募与分账规则 | 与拯救者/咨询师角色衔接 |
| P-204 | 勋章与成就体系统一升级 | 跨剧本、跨赛季 |

### 5.4 L3 — 心境与生态（12～24 个月+）

| 需求 ID | 需求描述 | 备注 |
|---------|----------|------|
| P-301 | **心境**：科普专栏、情绪自评工具（非诊断）、危机资源与热线醒目位 | 法务与医疗广告合规 |
| P-302 | 持证咨询师 / 机构入驻与预约导流 | 平台不接诊，仅信息与订单撮合（若涉及） |
| P-303 | **拯救者剧本**：咨询师参与设计的深度成长线 | 与 L0 引擎共用或扩展 Schema |
| P-304 | 组队闯关、活动赛季（可选） | 运营向 |

## 6. 非功能需求（平台级）

### 6.1 安全与合规

- **心理伦理**：全站明确「非医疗、非诊断」；心境模块突出危机干预入口。  
- **数据**：最小必要收集；档案上云后加密与可删除（个人信息保护法对齐）。  
- **UGC**：先审后发或机审 + 人工；禁止真人骚扰、封建迷信医疗承诺等。

### 6.2 性能与可用性

- 叙事核心保持 **首屏与关键路径** 在弱网可玩；音频策略继续遵守浏览器自动播放策略。  
- 平台期 SLA 按业务线分档（游戏可略低于支付与预约）。

### 6.3 多端

- MVP 已覆盖 **移动浏览器 + 桌面**；平台期原生 App 或小程序视渠道策略另立专章。

## 7. 技术架构演进方向（需求层）

| 阶段 | 架构取向 |
|------|----------|
| L0 | 纯前端 SPA + 静态剧本 JSON + localStorage（已实现） |
| L1 | 引入 BaaS 或自建轻后端：用户、档案同步、AI 网关 |
| L2 | 对象存储与 CDN 承载 UGC 素材；审核流水线；社区服务 |
| L3 | 心境与预约对接第三方 SaaS；合规日志与审计 |

*详细接口与表结构在单独《技术架构》迭代中落地。*
