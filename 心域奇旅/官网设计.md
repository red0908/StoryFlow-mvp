# 「心域奇旅 · MindScape」官网规划设计文档

**版本**：v1.0  
**日期**：2026-03-18  
**状态**：规划草案  

---

## 1. 引言

### 1.1 官网背景
「心域奇旅」是一款以 MBTI 人格理论为核心的轻量级相亲互动游戏，其最小可行性产品（MVP）已于 2026 年 3 月完成开发并上线测试，验证了“创建角色并用其进行沉浸式互动叙事”的核心玩法。为了将 MVP 成果与产品更宏大的愿景——“一个融合叙事、游戏与专业心理引导的数字化心灵游乐场”——相融合，同时向潜在用户、合作伙伴及投资者传递完整的品牌理念，我们需要建立一个概念介绍官网。

### 1.2 官网目标
- **品牌展示**：清晰传达“心域奇旅”的愿景、核心价值和差异化优势。
- **产品引导**：向访客介绍已上线的 MVP 版本，提供试玩入口，吸引种子用户。
- **生态预告**：展示未来平台蓝图（角色工坊、社区、心境等模块），建立用户期待。
- **信任建立**：通过团队介绍、设计理念等增强可信度，为后续融资、合作打下基础。

---

## 2. 官网定位与目标用户

### 2.1 定位
一个兼具“产品说明书”与“品牌故事”气质的轻量级营销官网，风格温暖、治愈、富有探索感，与游戏内体验一脉相承。

### 2.2 目标用户
| 用户类型 | 特征 | 访问目的 |
|----------|------|----------|
| 核心玩家（探索者） | 18-35 岁，对 MBTI、心理成长、互动叙事感兴趣 | 了解游戏玩法，尝试体验，加入社区 |
| 潜在创作者（引导者） | 心理咨询师、剧本作者，寻求新的创作与收入渠道 | 了解平台创作机制，考虑入驻 |
| 合作伙伴/投资人 | 关注心理、游戏、社交赛道 | 评估项目潜力，寻求合作机会 |
| 媒体/行业观察者 | 游戏媒体、心理类自媒体 | 获取产品信息，撰写报道 |

## 3. 信息架构与页面结构

官网采用单页滚动与多级页面结合的方式，核心信息集中在首页，次级页面提供深度内容。

```
首页 (Landing Page)
├── 导航栏 (Logo + 菜单：产品、生态、关于、试玩)
├── Hero 区域 (品牌Slogan + 核心价值 + 主视觉 + 试玩按钮)
├── 产品核心亮点 (MBTI + 双数值 + 四结局 + 沉浸音频)
├── 玩法速览 (剧情截图/动图，简要流程)
├── 平台生态 (角色工坊、社区、心境 模块预告)
├── 用户见证 (内测评价、数据)
├── 团队/关于我们
├── 底部 (版权、联系方式、社交媒体)
└── 试玩浮窗 (固定入口)

次级页面 (可选)：
- 产品详情页：详细说明游戏机制、剧本引擎、角色创建等。
- 生态蓝图页：详细介绍角色工坊、社区、心境等模块的未来规划。
- 新闻动态页：发布版本更新、活动、媒体报道。
- 关于我们页：团队介绍、愿景、联系方式。
```

## 4. 页面详细设计

### 4.1 导航栏
- 左侧：Logo + 产品名称「心域奇旅 · MindScape」
- 右侧：产品、生态、关于、**立即试玩**（按钮突出）
- 背景：半透明毛玻璃效果，滚动时变色。

### 4.2 Hero 区域
- **主视觉**：采用动态插画或循环短视频，展示角色在咖啡店、公园等场景互动的温馨画面，突出 MBTI 元素（如四种人格的抽象图标）。
- **大标题**：在另一个故事里，遇见自己。
- **副标题**：一款以 MBTI 为核心的轻互动叙事游戏。创建你的角色，开启一段 10 分钟的心灵奇旅。
- **CTA 按钮**：开始奇旅（指向 MVP 游戏入口）、观看预告片（可选）。
- **背景音乐**：自动播放主菜单 BGM（静音开关），营造沉浸氛围。

### 4.3 产品核心亮点
采用卡片式布局，每个亮点配图标和简短说明：
- **MBTI 人格碰撞**：基于真实人格理论，你的每一次选择都会影响对方的好感与自我认同。
- **双数值驱动**：好感度（对方感受）与贴合度（自我认同）共同决定四种结局——灵魂伴侣、表面和谐、知己难求、分道扬镳。
- **沉浸式音频**：场景专属 BGM、细腻音效，让每一次对话都身临其境。
- **角色档案**：每一次相遇都会被记录，在心域地图中收集你的 MBTI 角色图鉴。

### 4.4 玩法速览
以横向滚动截图或 GIF 展示关键页面：
- 剧本大厅 → 角色创建 → 选择相亲对象 → 剧情选项 → 结局页 → 心域地图
- 每张图配一句话说明，并添加“立即体验”小按钮。

### 4.5 平台生态（未来展望）
用“即将到来”标签区分，展示更宏大的蓝图：
- **角色工坊**：创造并训练属于你自己的角色 IP，定义它的 MBTI、背景故事、核心信念。
- **交流社区**：分享感悟、创作角色故事，与同好互动。
- **心境**：专业心理服务导航，为深度需求提供安全出口。
- **拯救者 / 探索者剧本**：更多元化的剧本类型，组队闯关与咨询师共创。

### 4.6 用户见证
- 展示内测玩家语录（匿名+MBTI 标签），如：“作为 INTJ，我居然在游戏里被感动到了。” —— 匿名玩家
- 数据亮点：MVP 上线 X 天，已有 Y 次心灵奇旅，Z 种结局被解锁。

### 4.7 关于我们
- 团队介绍：简短说明创始团队背景（心理、游戏、设计复合背景），配照片或插画。
- 愿景：打造一个安全、有趣、有温度的自我探索空间。
- 联系方式：邮箱、社交媒体链接（微博、小红书、Discord 等）。

### 4.8 底部
- 版权信息
- 隐私政策 / 用户协议
- 友情链接（如有）
- 回到顶部按钮

### 4.9 试玩浮窗
固定在右下角，显示“开始奇旅”按钮，点击直接跳转至 MVP 游戏入口（如 Web 游戏链接）。

## 5. 视觉与交互风格建议

### 5.1 视觉风格
- **主色调**：温暖治愈的渐变色，例如从淡紫到浅橙（对应 MBTI 的多样性与情感温度），辅以柔和的蓝绿色点缀（代表理性与平静）。
- **字体**：标题使用圆润有亲和力的无衬线字体（如思源黑体、Montserrat），正文保持清晰易读。
- **插画风格**：扁平化且富有细节的人物与场景插画，角色可带有 MBTI 性格特征的小元素（如 INTJ 的眼镜、ENFJ 的温暖笑容）。
- **动效**：微交互（按钮悬停、卡片浮现）使用柔和缓动，滚动时视差效果增强层次感。

### 5.2 交互建议
- 所有按钮都有悬停反馈。
- 截图可点击放大或播放视频。
- 在介绍双数值时，可用微型进度条演示其变化。
- 页面滚动时，导航栏透明度变化，CTA 按钮始终可见。

### 5.3 响应式适配
- 必须完美适配手机、平板、桌面端，尤其确保试玩入口在移动端易于点击。

## 6. 技术实现建议

### 6.1 技术栈
- **前端框架**：React / Vue 均可，配合静态站点生成（如 Next.js 或 Nuxt）以获得最佳 SEO 和性能。
- **样式**：Tailwind CSS 或 SCSS 模块化，便于快速构建响应式布局。
- **动效**：Framer Motion（React）或 GSAP 实现高级动画。
- **部署**：Vercel / Netlify 持续集成，支持一键回滚。

### 6.2 与 MVP 的集成
- 官网与游戏可独立部署，但保持同域名或子域名（如 `mindscape.game` 与 `mindscape.game/play`）。
- 官网的“立即试玩”按钮直接链接至游戏入口，需确保游戏已适配移动端且无自动播放音频拦截问题。

### 6.3 SEO 优化
- 每个页面配置元标签（标题、描述、关键词），利用 SSR 生成静态内容。
- 结构化数据（如产品、游戏）可提高搜索引擎展示效果。

### 6.4 性能考量
- 图片使用 WebP 格式，懒加载。
- 主视觉视频使用低码率压缩，或提供静音自动播放选项。
- 首屏加载时间控制在 2 秒内。

## 7. 上线与维护计划

### 7.1 上线准备
- **内容确认**：收集最终版文案、截图、视频素材，确保 MVP 体验链接稳定。
- **兼容性测试**：主流浏览器（Chrome、Safari、Edge）及移动设备。
- **备案/域名**：如有国内服务器需完成备案。

### 7.2 版本迭代
- **v1.0**：基础品牌展示 + MVP 引导（预计 1 周开发，2 周测试）。
- **v1.1**：增加新闻动态板块，发布首篇“研发故事”或“玩法解析”。
- **v2.0**：随着新模块（如角色工坊）开发，官网同步更新生态蓝图页面。

### 7.3 推广配合
- 官网发布时同步在社交媒体（微博、小红书、TapTap）预热，引导用户访问。
- 在游戏内增加“关于我们”或“官网”链接，形成双向导流。

## 8. 风险评估与应对

| 风险 | 应对 |
|------|------|
| 官网与游戏风格不一致 | 采用同一套设计语言（插画、配色、字体），由同一位设计师统筹。 |
| 试玩入口转化率低 | 在 Hero 区域和浮窗重复强调，并用动效吸引点击。 |
| 移动端体验卡顿 | 优化图片资源，减少动画复杂度，进行真机测试。 |
| 未来模块规划变更 | 官网生态部分使用“规划中”标签，并预留更新空间。 |
